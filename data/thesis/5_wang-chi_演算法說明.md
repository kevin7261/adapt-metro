## 計算邏輯與程式骨架

> 以下依論文寫「要算什麼、怎麼算」——用虛擬碼、公式與迴圈描述計算步驟。實作時照此邏輯寫；不要呼叫既有程式裡的函式名，也不要寫「call xxx」。

### 1. 輸入（抽象資料）

- 無向圖 `G = (V, E)`，每站 `v_i ∈ V` 帶地理座標 `(x_i, y_i)`（以螢幕像素或地圖單位表示）。
- 路線集合 `lines`：每條路線是站的有序序列；同一站可屬多條路線；focal edge 集合（Focus+Context 情境下的「目標路徑邊集合」）。
- 螢幕邊界 `[x_min, x_max] × [y_min, y_max]`。
- 算法參數：`w_g = 0.05, w_ℓ = 5, w_o = 10`，最小站-邊距離 `ε`，收斂容忍 `tol`。

### 2. 輸出

- 階段一輸出 `V′ = {v′_i}`：平滑化後的連續空間座標（邊長規律、角度均分，但方向任意）。
- 最終輸出 `Ṽ = {ṽ_i}`：八方向化後的座標（每條邊盡量水平/垂直/45°）。

### 3. 建議內部狀態

| 變數 | 說明 |
|---|---|
| `θ_ij` | 邊 `(i,j)` 的當前旋轉角（Gauss-Newton 外迴圈中交替更新） |
| `R_ij` | 旋轉矩陣 `[cosθ -sinθ; sinθ cosθ]`，初始為 `I` |
| `V′[i]` | 每站當前座標，在外迴圈中持續更新 |
| `D_α, D_β` | focal/contextual 目標邊長（由式 13 預先解出） |
| `cross_whitelist` | 原圖中已存在的邊交叉對（開始就有，不算違規） |
| `active_set` | 目前活躍的邊界/邊距懲罰項（只有違規時才加入） |
| `coarse_removed` | 粗化時依序移除的站記錄（含比例 r_i，供還原用） |

### 4. 主計算流程

```
──────────────────────────────────────────────────
前處理 A：解出目標邊長 D_α, D_β
──────────────────────────────────────────────────
// 式 13：總長守恆 + D_α = 2 D_β
// N_α = focal 邊數；N_β = contextual 邊數
// total_len = Σ |v_i - v_j|（所有邊的原始長度之和）
// 解方程組：
//   N_α * D_α + N_β * D_β = total_len
//   D_α = 2 * D_β
// → D_β = total_len / (2*N_α + N_β)
//   D_α = 2 * D_β
// 若無 focus（一般佈局），令 D_α = D_β = total_len / |E|

D_β ← total_len / (2*N_α + N_β)
D_α ← 2 * D_β

──────────────────────────────────────────────────
前處理 B：記錄交叉白名單
──────────────────────────────────────────────────
cross_whitelist ← {}
for each pair of edges (i,j), (k,l) where {i,j} ∩ {k,l} = ∅：
  if segments_intersect(v_i,v_j, v_k,v_l)：
    cross_whitelist.add({(i,j),(k,l)})

──────────────────────────────────────────────────
前處理 C：Coarse-to-fine 粗化
──────────────────────────────────────────────────
G_c ← G 的拷貝；coarse_removed ← []
repeat：
  best_v ← none；best_d ← ∞
  for each v_i that is a degree-2 in G_c（兩鄰居 v_j, v_k）：
    d_ijk ← point_to_segment_dist(pos[v_i], pos[v_j], pos[v_k])
    if d_ijk < best_d：best_d ← d_ijk；best_v ← v_i
  if best_d > threshold（e.g. ε·2）：break
  v_i ← best_v；兩鄰居 v_j, v_k
  r_i ← dot(pos[v_i]-pos[v_j], pos[v_k]-pos[v_j]) / |pos[v_k]-pos[v_j]|²
  coarse_removed.prepend((v_i, v_j, v_k, r_i))  // prepend 供還原時反序使用
  把 v_i 從 G_c 移除；若 (v_j,v_k) 不存在則新增邊 {v_j,v_k}
until 收斂或達到目標粗化大小

──────────────────────────────────────────────────
主迭代：階段一（平滑變形）——對 G_c 跑 Gauss-Newton 外迴圈
──────────────────────────────────────────────────
// 初始化
V′ ← G_c 的地理座標
R_ij ← I（單位矩陣）對所有邊
active_set ← {}

repeat（Gauss-Newton 外迴圈）：

  // ── 建超定線性系統 A·x = b（x = [v′_0.x, v′_0.y, v′_1.x, v′_1.y, ...] 展開） ──

  // Ω_m 貢獻（角度均分 + 線直，對每站 v_i、每對相鄰邊 {i,j},{i,k}）：
  //   令 f = deg(v_i)；θ_target = 2π / f
  //   ─── f = 2（直通站）────────────────────
  //   方程：v′_i = (v′_j + v′_k) / 2
  //   → 線性行：1·v′_i.x − 0.5·v′_j.x − 0.5·v′_k.x = 0（x 方向）
  //               1·v′_i.y − 0.5·v′_j.y − 0.5·v′_k.y = 0（y 方向）
  //   ─── f ≥ 3（轉乘站）──────────────────────
  //   等腰三角形幾何：v_i 應在底邊 v_j→v_k 的「中點法線」上，高度為
  //     h = tan((π − θ_target)/2) · |底邊|/2
  //   用上一輪 V′ 算 h（作常數），展開成 v′_i, v′_j, v′_k 的線性方程（見關鍵子計算 §5.1）
  //   每對 {i,j},{i,k} 貢獻 2 行（x, y 各一）

  // Ω_ℓ 貢獻（邊長規律，對每條邊 (i,j)）：
  //   s_ij = D_{ij} / |v_i - v_j|   （D_{ij} = D_α 或 D_β）
  //   目標：v′_i − v′_j = s_ij · R_ij · (v_i − v_j)
  //   令 target_ij = s_ij · R_ij · (v_i − v_j)（已知常數向量）
  //   行：1·v′_i.x − 1·v′_j.x = target_ij.x（帶權 sqrt(w_ℓ)）
  //       1·v′_i.y − 1·v′_j.y = target_ij.y

  // Ω_g 貢獻（錨定，對每站 v_i）：
  //   行：1·v′_i.x = v_i.x（帶權 sqrt(w_g)）
  //       1·v′_i.y = v_i.y

  // Active set 貢獻（邊界/邊距懲罰）：
  //   對每個 active 的邊界違規 v_i（e.g. v′_i.x < x_min）：
  //     行：1·v′_i.x = x_min（帶大權）
  //   對每個 active 的站-邊違規 (v_i, {j,k})：
  //     // p_jk = r·v′_j + (1-r)·v′_k（用上輪 V′ 算 r，作常數）
  //     δ_vec = ε / |v_i − p_jk| * (v_i − p_jk)（用上輪座標算，作常數）
  //     目標：v′_i − p_jk = δ_vec  → 展開成 v′_i, v′_j, v′_k 的線性行

  // ── 共軛梯度求解（CG）──
  V′_new ← CG_solve(AᵀA, Aᵀb)  （見關鍵子計算 §5.2）
  ΔV′ ← V′_new − V′

  // ── 交叉抑制（crossing damping）──
  for t = 1 to 20：
    new_cross ← false
    for each edge pair (i,j),(k,l) ∉ cross_whitelist：
      if NOT intersect(V′+(i,j)) AND intersect((V′+ΔV′)+(i,j))：
        new_cross ← true；break
    if NOT new_cross：break
    ΔV′ ← ΔV′ / 2
  V′ ← V′ + ΔV′

  // ── 更新旋轉矩陣 R_ij ──
  for each edge (i,j)：
    // 從「原始邊向量」旋轉到「當前邊向量」的角度
    θ_ij ← atan2(V′[j].y−V′[i].y, V′[j].x−V′[i].x)
          − atan2(v_j.y−v_i.y, v_j.x−v_i.x)
    R_ij ← [cos θ_ij, −sin θ_ij; sin θ_ij, cos θ_ij]

  // ── 更新 Active set ──
  for each v_i：
    if V′[i].x < x_min or V′[i].x > x_max or V′[i].y < y_min or V′[i].y > y_max：
      加入邊界懲罰 to active_set
    for each edge (j,k)（v_i 不在邊上）：
      if point_to_segment_dist(V′[i], V′[j], V′[k]) < ε：
        計算 r = dot(V′[i]−V′[j], V′[k]−V′[j]) / |V′[k]−V′[j]|²；clamp(r,0,1)
        加入站-邊懲罰（含 r 常數）to active_set

until ‖ΔV′‖ < tol 或達到最大迭代次數

──────────────────────────────────────────────────
粗到細還原：把 coarse_removed 的站逐一插回
──────────────────────────────────────────────────
for each (v_i, v_j, v_k, r_i) in coarse_removed（反序，最後移除的先還原）：
  V′[i] ← V′[j] + r_i * (V′[k] − V′[j])  // 線性內插回
  把 v_i 加回圖（G_c 擴充一個站）
// 以還原後的 V′ 繼續跑上述主迭代直到收斂（以細圖為目標）

──────────────────────────────────────────────────
階段二：八方向化（octilinearity）
──────────────────────────────────────────────────
// 以階段一最終 V′ 為輸入，建新系統
Ṽ ← V′

// Ω_o 貢獻（對每條邊 (i,j)）：
//   edge_vec = V′[i] − V′[j]
//   nearest_oct = 最近八方向單位向量（0°/45°/90°/135° 的正負向，共 8 個）
//   target_vec = nearest_oct * |edge_vec|  // 保長度，吸方向
//   行：1·ṽ_i.x − 1·ṽ_j.x = target_vec.x（帶權 sqrt(w_o)）
//       1·ṽ_i.y − 1·ṽ_j.y = target_vec.y

// Ω_g 貢獻（錨定到 V′，不再用原始座標）：
//   行：1·ṽ_i.x = V′[i].x（帶權 sqrt(w_g)）

// 注意：階段二不再有 Ω_m、Ω_ℓ 的行
// Active set（邊界/邊距）沿用階段一的方式動態加入

Ṽ_new ← CG_solve(A'ᵀA', A'ᵀb')
ΔṼ ← Ṽ_new − Ṽ
// 同樣做交叉抑制（damping），套用 ΔṼ
Ṽ ← Ṽ + ΔṼ

輸出 Ṽ
```

### 5. 關鍵子計算

#### 5.1 Ω_m 角度均分（f ≥ 3 展開）

```
站 v_i，一對鄰邊 {i,j},{i,k}，全角均分目標角 θ = 2π/f：
  // 等腰三角形：底邊從 v′_j 到 v′_k
  //   底邊中點 M = (v′_j + v′_k) / 2
  //   底邊向量 d = v′_k − v′_j
  //   法向（逆時針旋轉 90°）：n = (-d.y, d.x)
  //   v_i 應在 M + h_coeff * n 處，其中
  //   h_coeff = tan((π − θ)/2) / 2（h = tan((π−θ)/2) · |d|/2，除以 |d| 後只剩係數）

// 用「上一輪 V′ 的座標」計算 |d|，令 h_const = tan((π−θ)/2) * |d_prev| / 2：
  方程（x 方向）：
    v′_i.x = (v′_j.x + v′_k.x)/2 + h_coeff_prev * (-(v′_k_prev.y - v′_j_prev.y))
           = 0.5*v′_j.x + 0.5*v′_k.x + h_const_x
  其中 h_const_x = h_coeff * (-(v′_k_prev.y - v′_j_prev.y))（上一輪的常數）
  整理成：1·v′_i.x − 0.5·v′_j.x − 0.5·v′_k.x = h_const_x
  y 方向同理（n.y = d.x 分量）

// 特例 θ = π（f=2）：tan(0) = 0 → h_const = 0 → 退化成共線條件
```

#### 5.2 共軛梯度法（CG）骨架

```
// 求解 min ‖A·x − b‖²，即 AᵀA·x = Aᵀb
// 不顯式建 AᵀA；只需「矩陣×向量」乘積

CG_solve(A, b, x_init)：
  x ← x_init
  r ← b − A·x；p ← r；rs_old ← rᵀr
  for iter = 1 to CG_MAX：
    Ap ← A·p                   // 一次矩陣×向量
    α ← rs_old / (pᵀ·Ap)
    x ← x + α·p
    r ← r − α·Ap
    rs_new ← rᵀr
    if sqrt(rs_new) < tol：break
    β ← rs_new / rs_old
    p ← r + β·p
    rs_old ← rs_new
  return x

// 實務上把 AᵀA 理解為：
//   (AᵀA)·x = Aᵀ(A·x)
//   先算 z = A·x，再算 Aᵀ·z
//   — A·x：把未知數代入所有方程式右手邊（稀疏乘）
//   — Aᵀ·z：把每行的殘差反向累加到對應未知數
```

#### 5.3 交叉抑制（段相交判斷）

```
segments_intersect(p1, p2, p3, p4)：
  // 以向量叉積判斷 p1p2 與 p3p4 是否真正相交（不含端點接觸）
  d1 = cross(p4−p3, p1−p3)
  d2 = cross(p4−p3, p2−p3)
  d3 = cross(p2−p1, p3−p1)
  d4 = cross(p2−p1, p4−p1)
  if (sign(d1)≠sign(d2)) AND (sign(d3)≠sign(d4))：return true
  // 共線端點接觸：依實際需要決定是否計入（一般不算交叉）
  return false
```

## 1. 問題定義

輸入：地理座標的捷運圖 `G = {V, E}`（v_i ∈ ℝ² 為站的地理位置），以及（focus+context 情境下）一條由最短路徑演算法算出的「乘客路線」。
輸出：變形後的佈局——路線上的邊（**focal edges**）放大、其餘邊（**contextual edges**）縮短，整張圖仍塞進螢幕邊界；同時滿足捷運圖特性：線直、站距規律、交會角最大、八方向。

本法的核心定位：把所有幾何準則寫成**二次能量項**，用**最小平方（least squares）**解未知站位——連續最佳化、矩陣解法、**即時**（百餘站 < 1 秒），與 MILP（小時級離散最佳化）形成速度/品質的取捨。若把所有邊都設同一目標長度，即退化為**一般用途的示意圖佈局器**（論文圖 10 與 Nöllenburg 比較）。

### 兩階段策略（為什麼不一次解完）

八方向是**離散**性質，直接塞進連續目標函數會讓問題高度非線性。所以：

```
階段一（smooth deformation）：解出平滑佈局 V′——邊長規律化、夾角最大化、
                              保拓撲、進邊界，但方向還是任意的。
階段二（octilinearity）：把 V′ 的每條邊「就近吸附」到八方向，
                          再解一次最小平方得最終座標 Ṽ。
```

## 2. 階段一：平滑變形的能量項

未知數是新站位 `V′ = {v′_0 … v′_n}`。

### 2.1 Ω_ℓ 規律邊長（式 1）

每條邊變形後應等於目標長度（focal 邊 D_α、contextual 邊 D_β，D_α > D_β；論文設 **D_α = 2·D_β**）：

```
Ω_ℓ = Σ_{(i,j)∈E} | (v′_i − v′_j) − s_ij·R_ij·(v_i − v_j) |²
s_ij = D / |v_i − v_j|          # 縮放到目標長度 D ∈ {D_α, D_β}
R_ij = [cosθ_ij  −sinθ_ij; sinθ_ij  cosθ_ij]   # 未知的邊旋轉
```

- `s_ij` 把原邊縮放到目標長度；`R_ij` 允許邊**旋轉**（線要能拉直就必須允許轉）。
- θ_ij 是未知數 → 非線性，解法見 §4。
- 目標長度由式 13 決定：`N_α·D_α + N_β·D_β = Σ|v_i − v_j|`（總長守恆），加上 D_α = 2D_β 兩式解出 D_α、D_β。一般用途版（無 focus）令全部邊同一 D。

### 2.2 Ω_m 交會角最大化 + 線平直（式 2–3）

站 v_i 上共享 v_i 的每對相鄰邊 {i,j}, {i,k}，夾角應設為 **θ = 2π/f**（f = 交會邊數，均分全角）。假設兩邊等長，{i,j,k} 構成等腰三角形，可用相對位置寫成線性式：

```
Ω_m = Σ_{v′_i} Σ_{{i,j},{i,k}∈N(i)} | v′_i − ( u′_jk + tan((π−θ)/2)·⊥(u′_jk) ) − v′_j |²
u′_jk = ½ (v′_k − v′_j)        # ⊥ = 旋轉 90°
```

（實作上：v′_i 應位於底邊中點 + 沿底邊法向抬高 tan((π−θ)/2)·|底邊|/2 的位置——把這個幾何關係展開成對 v′_i, v′_j, v′_k 的線性方程即可。）

**特例 f = 2（一般中間站）**：θ = π，tan((π−θ)/2) = 0，退化成**共線約束**：

```
Ω_m = | v′_i − ½(v′_j + v′_k) |²        (式 3)
```

這一項就是「線平直」的來源——同一條線穿過度 2 站必須直走。

### 2.3 Ω_g 位置錨定（式 4）

```
Ω_g = Σ | v′_i − v_i |²
```

輕微把每站拉向原地理位置：(a) 固定整張圖的平移自由度；(b) 提供基本地理指引（北站不會跑到南邊）。權重要小。

### 2.4 合成目標與硬性條件（式 5）

```
min  Ω = Ω_m + w_ℓ·Ω_ℓ + w_g·Ω_g
s.t. x_min < v′_i,x < x_max,  y_min < v′_i,y < y_max     # 邊界
     v′_i − p′_jk ≥ ε   ∀ v′_i, {j,k}∈E                  # 站與不相鄰邊距離 ≥ ε
```

`p′_jk = r·v′_j + (1−r)·v′_k` 是邊 {j,k} 上離 v′_i 最近的點（r 由上一輪迭代的幾何算出、視為常數，這樣限制式對 v′ 是線性的）。

論文權重：**w_g = 0.05, w_ℓ = 5**（階段二 w_o = 10）。w_g 刻意小——只防漂移，不搶戲。

## 3. 階段二：八方向化（式 6）

對階段一結果 V′ 的每條邊，用函數 f 把邊向量**旋轉到最近的八方向**（0°/45°/90°/135° 軸），以此為目標再解一次：

```
Ω_o = Σ_{(i,j)∈E} | (ṽ_i − ṽ_j) − f(v′_i − v′_j) |²
min  Ω_g + w_o·Ω_o        （w_o = 10）
s.t. 邊界與邊距限制同式 5
```

注意：階段二**不再需要** Ω_m 與 Ω_ℓ——`f(v′_i − v′_j)` 本身已隱含了正確的長度與方向。

## 4. 求解（§3.3–3.4）——實作核心

### 4.1 最小平方骨架

把 Ω_m、Ω_ℓ、Ω_g 疊成超定線性系統 `A·V′ = b(V′)`，法方程解 `V′ = (AᵀA)⁻¹Aᵀb`。實際用**共軛梯度法（CG）**迭代解，不做 LU/Cholesky 分解——因為不等式限制會頻繁改動系統，直接分解太貴；CG 的核心只是矩陣×向量，也易平行化（論文考量行動裝置）。

### 4.2 Gauss–Newton 交替更新旋轉 R_ij

R_ij（θ_ij）是未知數 → 整體非線性。交替法：

```
初始化 R_ij ← I（單位矩陣）
repeat:
    固定所有 R_ij，解線性最小平方得 V′
    固定 V′，更新每條邊的 θ_ij ← 把 (v_i − v_j) 轉到 (v′_i − v′_j) 方向的角度
until 能量收斂
```

### 4.3 邊界處理（式 7）

每輪迭代檢查 v′_i 是否超出 [x_min,x_max]×[y_min,y_max]；超出的頂點**動態加入**懲罰能量：

```
Ω_b(v′_i) = |v′_i,x|²（x<0 時）、|v′_i,x − x_max|²（x>x_max 時）…（y 同理）
```

（把出界頂點拉回並沿邊界滑動。）

### 4.4 站–邊最小距離（式 8）

對太近的 (站 v′_i, 邊 {j,k}) 對，動態加入：

```
Ω_c = | (v′_i − p′_jk) − δ_{i→jk}·(v_i − p_jk) |²，  δ_{i→jk} = ε / |v_i − p_jk|
```

即把「站到邊」的向量拉長到至少 ε。

### 4.5 交叉抑制（crossing damping）——防止相對位置翻轉的關鍵

站–邊距離限制擋不住「四個端點都離很遠、但一步跨過去」的翻轉。所以：

```
每輪算出位移 ΔV′ 後：
    while 套用 ΔV′ 會產生「原圖沒有的邊交叉」：
        ΔV′ ← ΔV′ / 2        # 位移減半
    套用 ΔV′
```

原圖本來就存在的交叉（非平面網）不算違規。**這一步不可省**：省了會出現站的相對位置翻轉（Jacobi/Gauss-Newton 迭代的「翻滾」現象）；出現翻轉時的正確修法是實作/修好這個位移減半迴圈，**不是**去調能量權重。

### 4.6 Coarse-to-fine（§3.4）——防局部極小

不等式限制造成不連續 → 直接解容易卡局部極小。粗到細策略：

```
簡化：反覆移除「度 2 且 v_i 到線段 v_j v_k 距離最小」的站 v_i
      （移除後立即補上邊 {j,k} 防斷裂），直到最小距離超過門檻 → 得粗圖 G_c
求解：對 G_c 跑上述最佳化 → 粗結果 G′_c
還原：把移除的站按原相對比例線性內插回 G′_c 的邊上，作為全圖初始解
續解：以此初始解繼續最小化式 5 直到收斂
```

（雪梨 173 站 → 粗圖 52 站；表 1。）

## 5. 標籤（§4）——graph cuts

佈局完成後為 focal 站貼標（與佈局**解耦**以保即時性）。每站標籤位置 h_i ∈ 8 個八方向。能量：

| 項 | 型 | 內容 |
|---|---|---|
| Ψ_p 偏好 | data | 位置偏好成本：東/西 0（最好）、東北/西北 2、東南/西南 5、北/南 10（最差） |
| Ψ_q 標籤互疊 | smoothness | 兩標籤矩形相交 = 1，否則 0 |
| Ψ_b 出界 | data | 標籤矩形碰到螢幕邊界 = 1 |
| Ψ_e 寬敞度 | data | 把標籤矩形沿其八方向放大 2 倍得 qq_i，檢查是否撞到節點/邊：t = 最近交點距離/對角長；t<0.5 → 1；0.5<t≤1 → 1−2(t−0.5)；t>1 → 0 |
| Ψ_s 一致性 | smoothness | 相鄰站（有邊相連）標籤不同位置 = 1 |

總能量（式 12）：`Ψ = Σ(w_p Ψ_p + w_b Ψ_b + w_e Ψ_e) + Σ w_q Ψ_q + Σ w_s Ψ_s`，權重 **w_p = w_s = 1, w_q = w_e = 10, w_b = 20**（標籤絕不可出螢幕）。
用 **graph cuts（α-expansion）** 求解。技術細節：graph cuts 要求 smoothness 項滿足次模性 `Ψ(a,a)+Ψ(b,b) ≤ Ψ(a,b)+Ψ(b,a)`；令 Ψ_q(a,a) = Ψ_q(b,b) = 0 迴避（兩標籤同位置時靠 Ψ_e 寬敞度間接排開，因為兩站之間必有節點/邊）。

## 7. 實作檢查清單與陷阱

1. **兩階段不可合併**：八方向吸附放進階段一會把問題變成高度非線性/離散混合，收斂性崩壞。
2. **R_ij 初始化為單位矩陣**，交替更新；每輪從「原始邊向量 → 當前邊向量」重算 θ_ij，不是累積小角度。
3. **階段二丟掉 Ω_m、Ω_ℓ**，只留 Ω_g + w_o·Ω_o 與硬性條件。
4. **位移減半的交叉檢查**針對「新產生」的交叉；非平面網原有交叉要先記錄成白名單。
5. **p′_jk 的 r 用上輪座標算**，當常數 → 限制式保持線性。
6. 邊界/邊距是**違規才動態加項**（active set 風格），不是一開始全加——全加會讓系統又大又病態。
7. **coarse-to-fine 必做**（尤其大網）：直接細解會卡局部極小，結果歪斜。
8. 邊長會與拓撲衝突而偏離 D_α/D_β——正常現象，能量是軟性的（least squares sense，逼近不是強制）。
9. 一般用途版：所有邊同一目標長度、無 focal 加權，其餘照舊。
10. 已知限制（論文自承）：不維護「全域方位一致性」（整條線的絕對方向可能改變）；focal 邊太多且同向時會被螢幕寬度逼出多餘彎（圖 8）；標籤與佈局解耦，極端情況空間不夠仍會遮擋。
