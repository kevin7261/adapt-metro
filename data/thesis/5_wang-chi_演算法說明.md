# ⑤ Wang & Chi 最小平方能量法（Focus+Context 捷運圖）

> 論文：Yu-Shuen Wang & Ming-Te Chi (2011). *Focus+Context Metro Maps*. IEEE Transactions on Visualization and Computer Graphics, 17(12), 2011.

本文件是給開發者（或其他 LLM）的完整實作指南。讀完本文件後，不需要再看原始論文即可實作。

---

## 1. 問題定義

輸入：地理座標的捷運圖 `G = {V, E}`（v_i ∈ ℝ² 為站的地理位置），以及（focus+context 情境下）一條由最短路徑演算法算出的「乘客路線」。
輸出：變形後的佈局——路線上的邊（**focal edges**）放大、其餘邊（**contextual edges**）縮短，整張圖仍塞進螢幕邊界；同時滿足捷運圖特性：線直、站距規律、交會角最大、八方向。

本法的核心定位：把所有幾何準則寫成**二次能量項**，用**最小平方（least squares）**解未知站位——連續最佳化、矩陣解法、**即時**（百餘站 < 1 秒），與 MILP（小時級離散最佳化）形成速度/品質的取捨。若把所有邊都設同一目標長度，即退化為**一般用途的示意圖佈局器**（論文圖 10 與 Nöllenburg 比較）。

### 兩階段策略（為什麼不一次解完）

八方向是**離散**性質，直接塞進連續目標函數會讓問題高度非線性。所以：

```
階段一（smooth deformation）：解出平滑佈局 V′——邊長規律化、夾角最大化、
                              保拓撲、進邊界，但方向還是任意的。
階段二（octilinearity）：把 V′ 的每條邊「就近吸附」到八方向，
                          再解一次最小平方得最終座標 Ṽ。
```

## 2. 階段一：平滑變形的能量項

未知數是新站位 `V′ = {v′_0 … v′_n}`。

### 2.1 Ω_ℓ 規律邊長（式 1）

每條邊變形後應等於目標長度（focal 邊 D_α、contextual 邊 D_β，D_α > D_β；論文設 **D_α = 2·D_β**）：

```
Ω_ℓ = Σ_{(i,j)∈E} | (v′_i − v′_j) − s_ij·R_ij·(v_i − v_j) |²
s_ij = D / |v_i − v_j|          # 縮放到目標長度 D ∈ {D_α, D_β}
R_ij = [cosθ_ij  −sinθ_ij; sinθ_ij  cosθ_ij]   # 未知的邊旋轉
```

- `s_ij` 把原邊縮放到目標長度；`R_ij` 允許邊**旋轉**（線要能拉直就必須允許轉）。
- θ_ij 是未知數 → 非線性，解法見 §4。
- 目標長度由式 13 決定：`N_α·D_α + N_β·D_β = Σ|v_i − v_j|`（總長守恆），加上 D_α = 2D_β 兩式解出 D_α、D_β。一般用途版（無 focus）令全部邊同一 D。

### 2.2 Ω_m 交會角最大化 + 線平直（式 2–3）

站 v_i 上共享 v_i 的每對相鄰邊 {i,j}, {i,k}，夾角應設為 **θ = 2π/f**（f = 交會邊數，均分全角）。假設兩邊等長，{i,j,k} 構成等腰三角形，可用相對位置寫成線性式：

```
Ω_m = Σ_{v′_i} Σ_{{i,j},{i,k}∈N(i)} | v′_i − ( u′_jk + tan((π−θ)/2)·⊥(u′_jk) ) − v′_j |²
u′_jk = ½ (v′_k − v′_j)        # ⊥ = 旋轉 90°
```

（實作上：v′_i 應位於底邊中點 + 沿底邊法向抬高 tan((π−θ)/2)·|底邊|/2 的位置——把這個幾何關係展開成對 v′_i, v′_j, v′_k 的線性方程即可。）

**特例 f = 2（一般中間站）**：θ = π，tan((π−θ)/2) = 0，退化成**共線約束**：

```
Ω_m = | v′_i − ½(v′_j + v′_k) |²        (式 3)
```

這一項就是「線平直」的來源——同一條線穿過度 2 站必須直走。

### 2.3 Ω_g 位置錨定（式 4）

```
Ω_g = Σ | v′_i − v_i |²
```

輕微把每站拉向原地理位置：(a) 固定整張圖的平移自由度；(b) 提供基本地理指引（北站不會跑到南邊）。權重要小。

### 2.4 合成目標與硬性條件（式 5）

```
min  Ω = Ω_m + w_ℓ·Ω_ℓ + w_g·Ω_g
s.t. x_min < v′_i,x < x_max,  y_min < v′_i,y < y_max     # 邊界
     v′_i − p′_jk ≥ ε   ∀ v′_i, {j,k}∈E                  # 站與不相鄰邊距離 ≥ ε
```

`p′_jk = r·v′_j + (1−r)·v′_k` 是邊 {j,k} 上離 v′_i 最近的點（r 由上一輪迭代的幾何算出、視為常數，這樣限制式對 v′ 是線性的）。

論文權重：**w_g = 0.05, w_ℓ = 5**（階段二 w_o = 10）。w_g 刻意小——只防漂移，不搶戲。

## 3. 階段二：八方向化（式 6）

對階段一結果 V′ 的每條邊，用函數 f 把邊向量**旋轉到最近的八方向**（0°/45°/90°/135° 軸），以此為目標再解一次：

```
Ω_o = Σ_{(i,j)∈E} | (ṽ_i − ṽ_j) − f(v′_i − v′_j) |²
min  Ω_g + w_o·Ω_o        （w_o = 10）
s.t. 邊界與邊距限制同式 5
```

注意：階段二**不再需要** Ω_m 與 Ω_ℓ——`f(v′_i − v′_j)` 本身已隱含了正確的長度與方向。

## 4. 求解（§3.3–3.4）——實作核心

### 4.1 最小平方骨架

把 Ω_m、Ω_ℓ、Ω_g 疊成超定線性系統 `A·V′ = b(V′)`，法方程解 `V′ = (AᵀA)⁻¹Aᵀb`。實際用**共軛梯度法（CG）**迭代解，不做 LU/Cholesky 分解——因為不等式限制會頻繁改動系統，直接分解太貴；CG 的核心只是矩陣×向量，也易平行化（論文考量行動裝置）。

### 4.2 Gauss–Newton 交替更新旋轉 R_ij

R_ij（θ_ij）是未知數 → 整體非線性。交替法：

```
初始化 R_ij ← I（單位矩陣）
repeat:
    固定所有 R_ij，解線性最小平方得 V′
    固定 V′，更新每條邊的 θ_ij ← 把 (v_i − v_j) 轉到 (v′_i − v′_j) 方向的角度
until 能量收斂
```

### 4.3 邊界處理（式 7）

每輪迭代檢查 v′_i 是否超出 [x_min,x_max]×[y_min,y_max]；超出的頂點**動態加入**懲罰能量：

```
Ω_b(v′_i) = |v′_i,x|²（x<0 時）、|v′_i,x − x_max|²（x>x_max 時）…（y 同理）
```

（把出界頂點拉回並沿邊界滑動。）

### 4.4 站–邊最小距離（式 8）

對太近的 (站 v′_i, 邊 {j,k}) 對，動態加入：

```
Ω_c = | (v′_i − p′_jk) − δ_{i→jk}·(v_i − p_jk) |²，  δ_{i→jk} = ε / |v_i − p_jk|
```

即把「站到邊」的向量拉長到至少 ε。

### 4.5 交叉抑制（crossing damping）——防止相對位置翻轉的關鍵

站–邊距離限制擋不住「四個端點都離很遠、但一步跨過去」的翻轉。所以：

```
每輪算出位移 ΔV′ 後：
    while 套用 ΔV′ 會產生「原圖沒有的邊交叉」：
        ΔV′ ← ΔV′ / 2        # 位移減半
    套用 ΔV′
```

原圖本來就存在的交叉（非平面網）不算違規。**這一步不可省**：省了會出現站的相對位置翻轉（Jacobi/Gauss-Newton 迭代的「翻滾」現象）；出現翻轉時的正確修法是實作/修好這個位移減半迴圈，**不是**去調能量權重。

### 4.6 Coarse-to-fine（§3.4）——防局部極小

不等式限制造成不連續 → 直接解容易卡局部極小。粗到細策略：

```
簡化：反覆移除「度 2 且 v_i 到線段 v_j v_k 距離最小」的站 v_i
      （移除後立即補上邊 {j,k} 防斷裂），直到最小距離超過門檻 → 得粗圖 G_c
求解：對 G_c 跑上述最佳化 → 粗結果 G′_c
還原：把移除的站按原相對比例線性內插回 G′_c 的邊上，作為全圖初始解
續解：以此初始解繼續最小化式 5 直到收斂
```

（雪梨 173 站 → 粗圖 52 站；表 1。）

## 5. 標籤（§4）——graph cuts

佈局完成後為 focal 站貼標（與佈局**解耦**以保即時性）。每站標籤位置 h_i ∈ 8 個八方向。能量：

| 項 | 型 | 內容 |
|---|---|---|
| Ψ_p 偏好 | data | 位置偏好成本：東/西 0（最好）、東北/西北 2、東南/西南 5、北/南 10（最差） |
| Ψ_q 標籤互疊 | smoothness | 兩標籤矩形相交 = 1，否則 0 |
| Ψ_b 出界 | data | 標籤矩形碰到螢幕邊界 = 1 |
| Ψ_e 寬敞度 | data | 把標籤矩形沿其八方向放大 2 倍得 qq_i，檢查是否撞到節點/邊：t = 最近交點距離/對角長；t<0.5 → 1；0.5<t≤1 → 1−2(t−0.5)；t>1 → 0 |
| Ψ_s 一致性 | smoothness | 相鄰站（有邊相連）標籤不同位置 = 1 |

總能量（式 12）：`Ψ = Σ(w_p Ψ_p + w_b Ψ_b + w_e Ψ_e) + Σ w_q Ψ_q + Σ w_s Ψ_s`，權重 **w_p = w_s = 1, w_q = w_e = 10, w_b = 20**（標籤絕不可出螢幕）。
用 **graph cuts（α-expansion）** 求解。技術細節：graph cuts 要求 smoothness 項滿足次模性 `Ψ(a,a)+Ψ(b,b) ≤ Ψ(a,b)+Ψ(b,a)`；令 Ψ_q(a,a) = Ψ_q(b,b) = 0 迴避（兩標籤同位置時靠 Ψ_e 寬敞度間接排開，因為兩站之間必有節點/邊）。

## 7. 實作檢查清單與陷阱

1. **兩階段不可合併**：八方向吸附放進階段一會把問題變成高度非線性/離散混合，收斂性崩壞。
2. **R_ij 初始化為單位矩陣**，交替更新；每輪從「原始邊向量 → 當前邊向量」重算 θ_ij，不是累積小角度。
3. **階段二丟掉 Ω_m、Ω_ℓ**，只留 Ω_g + w_o·Ω_o 與硬性條件。
4. **位移減半的交叉檢查**針對「新產生」的交叉；非平面網原有交叉要先記錄成白名單。
5. **p′_jk 的 r 用上輪座標算**，當常數 → 限制式保持線性。
6. 邊界/邊距是**違規才動態加項**（active set 風格），不是一開始全加——全加會讓系統又大又病態。
7. **coarse-to-fine 必做**（尤其大網）：直接細解會卡局部極小，結果歪斜。
8. 邊長會與拓撲衝突而偏離 D_α/D_β——正常現象，能量是軟性的（least squares sense，逼近不是強制）。
9. 一般用途版：所有邊同一目標長度、無 focal 加權，其餘照舊。
10. 已知限制（論文自承）：不維護「全域方位一致性」（整條線的絕對方向可能改變）；focal 邊太多且同向時會被螢幕寬度逼出多餘彎（圖 8）；標籤與佈局解耦，極端情況空間不夠仍會遮擋。
