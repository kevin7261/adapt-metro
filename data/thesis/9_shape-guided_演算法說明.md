## 計算邏輯與程式骨架

> 以下依論文寫「要算什麼、怎麼算」——用虛擬碼、公式與迴圈描述計算步驟。實作時照此邏輯寫；不要呼叫既有程式裡的函式名，也不要寫「call xxx」。

### 1. 輸入（抽象資料）

- 運輸網 `N = (S, C)`：
  - `S = {v1,…,vn}`：車站，各附地理座標 `(x_i, y_i)`。
  - `C = {e1,…,em}`：連線（無向邊）。
- 路徑覆蓋 `T`：營運線集合，每條線是 `S` 上的路徑；每條邊 `e ∈ C` 屬一或多條線。
- 引導形狀 `P = {P1,…,Pk}`：一或多條開／閉折線（polyline），頂點序列任意幾何（愛心、圓、熊、眼睛…）。
- （選配）手動指定路徑 `W_manual`：藝術／廣告用途時跳過自動比對。
- 參數：dummy 距離門檻、變形權重 `wc, wl, wa, wp, wo`、格距因子 `fd`、Octi 成本 `chop`、接縫距離 `dmin, dlow, dup` 等（見 §建議參數）。

### 2. 輸出

混合式示意圖佈局：

- 描形邊 `C_shape`：貼著引導形狀走（方向任意）。
- 其餘邊 `C_octo`：嚴格八方向（0°/45°/90°/…）。
- 全部站與邊對齊格網（Octi 改造後的格點最短路）。

### 3. 建議內部狀態

| 變數 | 說明 |
|---|---|
| `W` | 比對路徑（站序列／邊序列），用來**擺位**形狀（≠ 描形邊集） |
| `P_aligned` | 經等比縮放＋平移後的形狀（凍結，變形不再動） |
| `S′`, `C′` | 平滑段當前站／邊位置 |
| `S′_shape`, `C′_shape` | 每輪動態指派的描形站／邊 |
| `S̃`, `C̃_shape`, `C̃_octo` | 混合段結果與邊分類 |
| `θ_ij`, `R_ij` | 邊旋轉角／旋轉矩陣（平滑段 Ωl 用） |
| `target_dir[e]` | 混合段每條 octo 邊的目標八方向角 |
| `G = (VG, EG)` | Octi 格網圖（含 port／sink／彎折邊）＋織入的形狀節點 |
| `Σ` | 格網路由順序（先 `C_shape` 後 `C_octo`） |
| `ψ[s]` | 站定案的格點位置 |
| `deg2_chains` | 收縮後待回插的 deg-2 站鏈 |

### 4. 主計算流程（三步管線）

```
──────────────────────────────────────────────────
前處理
──────────────────────────────────────────────────
若 N 非平面：在每對交叉邊插 dummy 站，斷邊成兩段
若 deg(v) > 8：把高密度站拆成多個 deg < 8 的站（八方向 port 上限）

──────────────────────────────────────────────────
STEP 1：路線比對與形狀擺位（§4）
──────────────────────────────────────────────────
if W_manual 有給：
  W ← W_manual
else：
  W ← AutomaticRouteMatching(N, P)   // 見關鍵子計算 §A

// 只做平移＋等比縮放，不旋轉（D5）
bbox_W ← bounding_box(W 的地理座標)
bbox_P ← bounding_box(P)
scale ← 依 bbox 對齊的等比縮放因子（保長寬比）
offset ← 使 P 中心對到 W 中心的平移
P_aligned ← scale * P + offset

若 P 有多條折線（如巴黎「眼睛」）：
  只對「含左上頂點」的那條折線跑比對；
  其餘折線用同一 scale／offset 直接放上去

// 之後 P_aligned 凍結，變形階段不得再動大小位置

──────────────────────────────────────────────────
STEP 2a：平滑佈局（§5.1）——最小平方迭代
──────────────────────────────────────────────────
平面化後的網路記為 (S, C)
V′ ← 各站地理座標（複製）
R_ij ← I（單位矩陣）對所有邊
L ← 初始佈局的平均邊長
            // dummy 站相連的邊目標長 = L/2；其餘 = L

repeat（Gauss-Newton／共軛梯度外迴圈，至能量收斂）：

  // ── 動態指派描形站（§5；W 不用來定 C_shape）──
  S′_shape ← {}
  for each station v′i ∈ S′：
    pi ← P_aligned 上距離 v′i 最近的點
    v*i ← v′i 對 pi 的鏡射點（pi 為中點：v*i = 2·pi − v′i）
    if 線段 v′i–v*i 不與任何邊 e ∈ C′ 相交：
      S′_shape.add(v′i)
      // 含義：v′i 與形狀之間沒有「更近的線」——由更近者負責描形
  C′_shape ← { 邊 {i,j} ∈ C′ | 兩端都在 S′_shape }

  // ── 建超定系統，最小化 Ω_smooth（式 1）──
  // Ω_smooth = wc·Ωc + wl·Ωl + wa·Ωa + wp·Ωp
  //
  // Ωc 貼形（式 2）：Σ_{v′i∈S′_shape} D(v′i)·|pi − v′i|²
  // Ωl 等邊長（式 3）：Σ_{{i,j}∈C} |(v′i−v′j) − s_ij·R_ij·(vi−vj)|²
  //                   s_ij = L_target / |vi−vj|；R_ij 旋轉矩陣（罰長度不罰轉向）
  // Ωa 角解析度（式 4）：對 deg>2 的站，以 tan((π−θi)/2) 的切線構造
  //                   把相鄰出邊撐開；θi = 2π/D(v′i)；deg=2 自然拉直
  // Ωp 定位（式 5）：Σ |v′i − vi|²（護心智地圖）
  //
  // 固定 R_ij，解線性最小平方得 V′；再固定 V′ 更新每邊 θ_ij／R_ij
  // （交替法同 ⑤ Wang & Chi）

  // ── 平面性守門（每輪必做）──
  if 出現新交叉：
    相關站退回上一輪位置
    加能量項懲罰同一交叉再犯
  if 站–邊距離 < 門檻：
    加能量項把站與邊推開

until 收斂

──────────────────────────────────────────────────
STEP 2b：混合佈局（§5.2）——近八方向＋貼形
──────────────────────────────────────────────────
// 邊分類承襲平滑段終態（不再重算鏡射）
C̃_shape ← C′_shape
C̃_octo  ← C′ \ C′_shape
Ṽ ← V′（平滑結果作初值）

// 為每條 octo 邊指派最近八方向斜率
for each edge e ∈ C̃_octo：
  target_dir[e] ← 最接近目前斜率的 k·π/4（k=0..7）

// 衝突（同一站多邊搶同一方向）→ 匈牙利演算法整體重指派
// 目標：最小化 Σ |目前角 − 指派角|（總旋轉量）
target_dir ← HungarianReassign(C̃_octo, target_dir)

repeat（最小平方外迴圈）：
  // Ω_mixed = wo·Ωo + wp·Ωp + wc·Ωc（式 7）
  // Ωo（式 6）：Σ_{{i,j}∈C̃_octo} |(θ′ij + Δθij) − θ̃ij|²
  //   θ′ij = 平滑段結束時的邊角；Δθij = 到目標八方向的差；θ̃ij = 當前角
  // Ωp、Ωc 同平滑段（描形站仍往 P_aligned 吸）
  解最小平方更新 Ṽ
  平面性守門（同 2a：交叉退回＋加懲罰；站–邊過近推開）
until 收斂

──────────────────────────────────────────────────
STEP 3：格網對齊（§6）——改造 Octi
──────────────────────────────────────────────────
輸入改為混合佈局 (S̃, C̃)，非原始地理座標

// 3a. deg-2 收縮（相對原 Octi 的例外）
G_n ← 網路拷貝；deg2_chains ← {}
for each deg-2 站 v（兩鄰 a,b）：
  if v ∈ S_shape：
    保留（D6：描形站全數保留）
  else if 兩鄰邊被指派到「不同且非相對」的八方向角：
    保留（混合段已依 D4 決定此彎是刻意的）
  else：
    收縮：記錄 deg2_chains；合併邊 {a,b}；移除 v
// 度 ≥3 的站一律保留（同原 Octi）

// 3b. 建格網 G（同 ⑥ Bast）
d ← 平均站距 × fd
建含對角的格網圖；每格點 = 1 sink + 8 port
port–sink 邊成本 csink；port 間彎折成本依轉角；跨格 hop 成本 chop

// 3c. 把 P_aligned 織進格網（論文圖 8）
P_sub ← 重取樣 P_aligned，使相鄰頂點距 ≈ d
         // 只加點不刪點，保留形狀特徵角
刪掉 G 中與 P_sub 相交的邊
刪掉與 P_sub 頂點距離 < dmin 的格點
for each 形狀頂點 pi ∈ P_sub：
  for each 格點 a，若 dlow ≤ |pi−a| ≤ dup：
    互連 pi ↔ a（接縫邊；不必然八方向——D6 > D7）

// 3d. 權重（誘導路徑走 P）
port–sink：形狀節點與一般格點相同
跨 sink 的 hop：
  一端在 P 上 → chop/2
  兩端都在 P 上 → chop/20
同 sink 的 port 彎折：成本用 2π − θ 一般化
  // P 非八方向、θ 任意；θ ≤ π 保證「兩段繞行不會比直連便宜」

// 3e. 路由順序與候選集
Σ ← C_shape 全部在前，再接 C_octo
for each 站 s：
  if s ∈ S_shape：
    V^c(s) ← 距離最近的 2 個格點   // 鎖死，不准亂跑
  else：
    V^c(s) ← 半徑 r 內的 sink（同原 Octi）

for each edge {s,t} in Σ：
  在 G 上跑 set-to-set 最短路（端點候選 V^c）
  定案路徑；佔用的 port／格點資源關閉
  // 已定案端點直接從定案格出發，不再加虛擬頂點

// 3f. 回插
把 deg2_chains 的站沿曲線等距插回
輸出最終格網佈局
```

## 關鍵子計算

### A. 自動路線比對（§4.1）——shape-preserving Dijkstra

目標：找路徑 `W` 使方向式積分 Fréchet 距離 `δ(W, P)` 最小，用來決定 P 的縮放與平移。

```
ALGORITHM AutomaticRouteMatching(N, P):

  // ── 補 dummy 邊 ──
  for each pair vi, vj ∈ S 且無邊相連：
    if |vi − vj| < dummy_threshold：
      插入 dummy 邊 {i,j}
      // dummy 邊成本「高」、真實邊成本「低」→ 偏好少用 dummy

  // ── 從每個候選起點成長路徑，取 δ 最小者 ──
  best_W ← none；best_δ ← ∞

  for each 候選起點 s ∈ S：
    W ← [s]                         // W0
    repeat：
      queue ← {}
      for each 鄰站 v of W 的端點：
        if δ(W + v, P) < δ(W, P)：   // 方向式積分 Fréchet
          queue.add(v)
      if queue 空：break

      // 在可改善候選中，取部分相似最小者延伸
      u* ← argmin_{u ∈ queue} δ_partial(W + u, P)
      // δ_partial(W,P) = min_{P′ ⊆ P} δ(W, P′)
      // （長路徑成長時只跟「目前最像的那一段」比）
      W ← W + u*
    until 佇列空

    if δ(W, P) < best_δ：
      best_δ ← δ(W, P)；best_W ← W

  return best_W
```

**為何用方向式 Fréchet（而非點距）**：比的是斜率差，天生**平移／縮放不變**——P 的大小位置尚未決定時，這是唯一合理的度量；小變異不會被過度懲罰（de Berg & Cook 2011；Funke & Storandt 2011 的 shape-preserving Dijkstra 改造版）。

### B. 描形站動態指派（平滑段每輪）

```
pi  = closest_point_on_polyline(v′i, P_aligned)
v*i = 2·pi − v′i                              // 對 pi 鏡射
若 segment(v′i, v*i) ∩ C′ = ∅：
  v′i ∈ S′_shape
否則：
  表示另有一條線夾在 v′i 與形狀之間 → 由那條線負責描形
```

**鐵律：`W ≠ C_shape`**。W 只抓整體輪廓、會漏細節；寫死用 W 當描形邊集會漏角或抓錯側的線。

### C. 混合段八方向衝突 → 匈牙利重指派

每條 octo 邊貪最近 `k·π/4` 會在同一站撞車（兩出邊要同一方向）。把「邊 ↔ 可用八方向槽」建成指派問題，成本 = 旋轉角絕對值，匈牙利法一次解出全域最小總旋轉。

### D. Ωa 角解析度（式 4）的幾何意義

對 deg = f ≥ 3 的站、每對相鄰出邊 `{i,j},{i,k}`：

```
θi = 2π / f
c′ij = (v′k − v′j) / 2                         // 底邊一半向量
目標：v′i ≈ v′j + c′ij + tan((π − θi)/2) · c′⊥  // 等腰高
```

deg = 2 時等價於「站在兩鄰中點」→ 自然拉直。

### E. Octi 接縫距離關係

```
dmin < dlow < d < dup
論文評估值：dmin = d/5，dlow = d/2，dup = 3d/2
```

接縫邊非八方向是**刻意取捨**（形狀可辨識 D6 優先於嚴格八方向 D7）。

## 限制（§8.3）

1. 路線比對是貪婪成長，**不保證全域最優**；多折線形狀無法對「哪些子段對辨識最重要」加權（整形等權）。
2. 變形是局部最小平方，形狀可能**卡住高密度區展開**（莫斯科圈偏小）。
3. 格網對齊：P–格網接縫**必然有非八方向邊**；大網可能有少量邊 route 不出來（東京心形 <5 條失敗）；分鐘級耗時。
4. 標籤是事後貼（Niedermann & Haunert），可能產生過小字。
5. 不是「任何形狀都適合捷運圖」——行銷／海報場景另論。

## 實作檢查清單與陷阱

1. **不旋轉是鐵律（D5）**：比對、對齊、變形全程只有平移＋等比縮放。
2. **方向式 Fréchet 而非點距**：比對度量必須平移／縮放不變，否則 P 未定大小位置會讓比對失義。
3. **W ≠ 描形邊集**：W 只拿來擺形狀；C_shape 每輪用「鏡射線段不穿邊」動態重算。
4. **dummy 邊要貴**：否則比對路徑會全走 dummy、跟真實路網脫鉤。
5. 變形每輪要做**平面性回退**（交叉→退回＋加懲罰），否則最小平方會安靜地把線疊穿。
6. 混合段八方向衝突**必須匈牙利整體重排**，不能各自貪最近角。
7. 格網路由順序：**先 shape 邊後 octo 邊**；描形站候選鎖最近 2 格——順序反了形狀會被一般邊搶走廊。
8. P 織入的三個距離（dmin/dlow/dup）決定接縫品質；接縫非八方向是刻意的（D6 > D7）。
9. 高度數站（>8）先拆站，否則八方向 port 不夠用。
10. 選型：要「嵌任意符號形狀」用本法；只要環線變正圓／多邊形，MILP（③ 有 loop line 特化）或手繪規則更省事。

## 與本系統（Adapt-Metro）的關係

**對照法，未實作對應章節**（使用者裁決不做）。本系統管線（骨架化 → 格網化 → 爬山＋三步鏈 → RWD 畫線）沒有「使用者指定形狀」輸入。概念親緣：

- 第三步格網對齊直接改造 **Bast et al.（⑥）**——與本系統 RWD 的 A\* 格點救援同源（port／彎折成本／逐邊定案）。
- 「描形邊放寬方向、其餘守八方向」的**混合方向**思想，與本系統 16 方向（22.5°/67.5° 當受控例外）異曲同工：都是「局部放寬角度換取全域品質」。
- 第二步變形沿用 **Wang & Chi（⑤）** 的兩段最小平方（先平滑再吸方向）。
- 若未來要做「把台北環狀線畫成正圓／愛心」之類需求，本文三步框架（比對 → 變形 → 對齊）是首選參考；LLM 指定對齊（`route-llm-align` 的 prompt 模式）可視為「以自然語言代替幾何形狀輸入」的輕量替身。
