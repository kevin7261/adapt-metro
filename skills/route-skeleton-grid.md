# 示意圖佈局（示意格網化）(route-skeleton-grid)

把 [[route-skeleton-connect]] 骨架（原始骨架化 或 旋轉骨架化）收成**整數格路網**：彩色點用藍色
分隔網格做**排名吸附**，每條路線在彩色邊界切開，黑點沿新邊平均放回。此 skill 是這個轉換的
**唯一權威依據**。

> **不重做骨架拓撲、不跨路線合併**；每條骨架邊各自處理。純幾何、零網路零 token。
> 實作：`src/stores/schematicGrid.js` 的 `buildSchematicGrid(skeleton, posById, extent)`；
> UI 在 `D3Tab.vue`（Map Adjust）的兩個 tab：**格網化前**（畫藍格預覽、座標未改）與
> **格網化後**（吸附後的整數格路網）；**兩者都在上面畫藍格**。

## 整體流程

```
骨架（線 + 站，來自骨架化）
  → ① 蒐集彩色點，用 x/y 排名建藍色分隔網格（欄、列）
  → ② 每條線在彩色邊界切開（同一條線內切、不跨線）
  → ③ 各小段端點吸附到整數格（欄, 列）
  → ③′ 吸附後修復（repairOcclusions）：消除排名吸附造成的 壓點/交叉/共線重疊
  → ④ 黑點沿新（拉直）小段平均放回
```

## 點的分類（沿用骨架的 `stationClass`）

**彩色點＝所有非黑的節點**：紅（分歧/轉乘）、藍（端點）、**黃（幾何交叉，[[route-skeleton-connect]]
的 `crossings`，已實作）**、紫（切斷）、粉紅（轉折）、灰（分隔）（＋棕，若有）。這些都是
**邊界**——排名與切線都用它們，**「一格至多一個彩色點」的限制黃色點也一體適用**。
（黃色點非車站：D3Tab 需把 `sk.crossings` 座標放進 `posById`，`buildSchematicGrid` 才數得到。）
**黑點（直通站）不切格、不佔欄列排名**。

## ① 排名吸附（藍色分隔網格）

- **最密切法（使用者指定）**：在每兩個相鄰**不同座標**之間都下一條分隔線——每個不同 x
  各自成一欄、每個不同 y 各自成一列。格網＝彩色點數量級（Taipei 67×67、Beijing 165×165、
  NYC 270×270）。這是**唯一能保證連近重合黃色交叉（間距僅 0.1~18m）也一格一點**的切法；
  粗切法（TOL=10）雖然格網較稀，但那些近重合點會落在同格，是**重大錯誤**，故改回最密切。
- 每個彩色點得到唯一 `(欄, 列)`（rank = 位置前的分隔線數，全不同 x ⇒ 每點獨佔一欄）。
  唯一例外：兩點座標**完全相同**（gap < 1e-6）→ 由撞格 bump（最近空格）分開，POST 0 碰撞。
- **PRE 與 POST 都保證每格 ≤1 彩色點**（含黃）。
- **硬性：一格至多一個彩色點（白/黑除外）**。若兩彩色點被排到同一格 → **撞格解衝突**：
  將後者外擴到**最近的空格**（Chebyshev 環狀搜尋，位移最小），保證每格 ≤1 彩色點。
- **藍線是「分隔線」——畫在格與格之間，絕不穿過點**；每個彩色點吸附到其格子的**中央**。
  （早期版把點放在格線交點上是錯的：視覺上會出現一格多點/點壓線。）
- **代價**：欄列是**等距排名**，不再保留真實地理距離比例（之後若有落線/交叉交給下游處理）。
- 藍格畫在**最底層**：**前**＝相鄰彩色點群中點畫分隔線（預覽，每個 rank 帶各自成一格）；
  **後**＝解衝突後的等寬格界（cols+1／rows+1 條線），格座標標在**格子中央**（下方欄、左方列）。

## ② 每條線在彩色邊界切開

對骨架每條路線折線：起點、終點一定是切點；中段頂點若是彩色點也是切點。
相鄰切點間成一段「小段」。**只在同一條線內切，不跨線合併。**

## ③′ 吸附後修復（repairOcclusions，大邱重疊案 2026-07）

排名吸附是**逐軸獨立的排名變換**，會扭曲相對幾何：地理上不共線的三點可能吸附後
恰好共線（**點壓到別的段上／兩段共線重疊**）、不相交的兩段可能吸附後**相交**
（掃描 2026-07：234 城中 45 城輸入有 1–10 個）。吸附＋撞格解衝突後、算 posAfter
前，跑修復迴圈：

- 段圖＝骨架每條邊在彩色切點切開的小段（與 placeBlacks／下游 buildHcGraph
  **同一套切法**；稀有黑切點先給格子，修復看到的圖與爬山法完全相同）。
- 每輪取第一個違規（壓點/交叉/共線重疊），依序試每個肇事點的環狀外移
  （Chebyshev 半徑 1–6、同半徑取位移最小、只進空格），接受第一個讓違規數
  **嚴格下降**的移動（局部計數：移動 v 只影響涉及 v 的違規）；全零或卡住為止
  （上限 200 步，卡住保持現狀、不劣化）。確定性、無亂數。
- 與撞格 bump 同性質：**犧牲一點排名位置換取正確性**。
- 下游爬山鏈的硬規則（[[route-hillclimb]] ③/③′＋compactGridSafe）保證「只減
  不增」⇒ **輸入全零＝格網化後→爬山→後處理→循環全程零 壓點/交叉/重疊**。

## ③④ 端點吸附 + 黑點均分

每一小段：兩端（彩色點）吸到整數格 `(欄,列)`；中間黑點沿「兩端整數格連線」**平均放回**
（第 j 個黑點在 j/(k+1) 處）。端點吸附與預覽藍格**同一套**排名，前後格線意義一致。

## 硬性原則

1. **黑點不切格、不佔欄列排名。**
2. **紅／藍／黃／紫／粉紅／灰（及棕）都是邊界**——線在這些點切開。
3. **每條 way 各自處理**，不跨 way 併邊改拓撲。
4. **起訖端點吸附與預覽藍格同一套排名**，開始執行前後格線意義一致。
5. **路線中間不該無故冒紅／藍**：那是骨架階段已分好的；⑨ 只平移到整數格並重放黑點。

## 一句話

彩色點排出藍格排名 → 路線在彩色點切開 → 端點吸到整數欄列並拉直 → 黑點在兩端之間平均插回。

## 實作契約

- `buildSchematicGrid(skeleton, posById, extent)` 為**純函式**：`skeleton` 為 connect 骨架、
  `posById` 為 `Map<id,[x,y]>`（已投影的螢幕座標，含旋轉）、`extent=[x0,y0,x1,y1]`。回傳
  `{ posAfter: Map<id,[x,y]>, blueBefore:{xs,ys}, blueAfter:{xs,ys}, cellOf: Map<id,[col,row]>, cols, rows }`
  ——`cellOf` 是每個切點的整數格，供下游 [[route-hillclimb]]（②爬山最佳化）使用。
- 排名用彩色點（`stationClass !== 'black'`）的 x/y 分群；黑點只由小段內插得到 posAfter
  （內插抽成 `placeBlacks(skeleton, posMap, snap)` 匯出，爬山法重放黑點也用它）。
- **撞格解衝突已實作**（`nearestFree`，Chebyshev 環狀外擴）——保證每格 ≤1 彩色點。
- 藍格畫在**最底層**（在線與站點之下）。落線/交叉矯正仍列為下游。改時同步本檔。

## 修改此轉換時

排名/切線/吸附規則變動，**同步更新本 SKILL.md 與 `src/stores/schematicGrid.js`**。
