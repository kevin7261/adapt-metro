04城市判定與合併CITY RESOLUTION

「同一城市＝同一系統」的三層防護

OSM 車站常無 network 標籤，城市名稱解析容易撞名（如 S-Bahn Frankfurt 的 bahn token 撞上 Berlin U-Bahn）——三層護欄避免抓錯城市。

1 · token 黑名單

泛用交通詞（bahn／ubahn／sbahn／lrt／tram／verkehrsverbund…）不得參與城市名稱比對。

2 · rebucket 距離護欄

member network 解析出的目標城市，須離群組重心 ≤250 km 才准搬家；全國性營運商（如 DB）重心無意義。

3 · 線級距離護欄

解析出的 wiki 城市離該線重心 >250 km ⇒ 改用線自身座標或原始反查城市，自成獨立系統檔。

都會圈合併 CITY_MERGE

鄰接行政市併成一城一檔（如桃園→台北）；被併城市在驗證時以 covered_by 標註通過。

LRT 範圍規則

附掛輕軌的城市（台北／高雄／新加坡）附加收錄；整系統即為 LRT tagging 者（Vancouver SkyTrain）全收；純附掛輕軌原則剔除，但覆蓋率 <0.6 時例外保留。

06資料格式 · 每城一檔DATA FORMAT · GEOJSON

一份 FeatureCollection：系統中繼資料＋兩種 feature

每座城市一個 GeoJSON（systems/{洲}/{國}/{洲2碼}-{IOC3碼}-{城}.geojson，如 as-twn-taipei.geojson）：頂層掛 metro_system 系統中繼資料，features 只有兩種——車站 Point 與 路段 MultiLineString。

{ "type": "FeatureCollection", "metro_system": { "continent": "asia", "country": "Taiwan", "city": "Taipei", "operator": "臺北大眾捷運股份有限公司", "official_website": "…", "wikidata": "Q…", "line_count": 18, "segment_count": 20, "station_count": 197, "audit": { …逐城驗證結果 } }, "features": [ { …Point 車站 }, { …MultiLineString 路段 } ] }
幾何語意 1

線永遠壓在站點上：路段幾何＝共站合併後的車站點依站序連線，每個折點、端點都是車站（跳站特快段例外，見 Stage 2）。

幾何語意 2

重疊路段只畫一條：相鄰車站對被多線共用時只輸出一個 feature，routes list 記行經路線——一條 route 的完整路徑＝所有含它的路段之聯集。

幾何語意 3

快慢車合併：同 network＋ref 的快車／區間車合併為一條線，以站最多的慢車站表為準；示意站序線形，非真實軌道線形。

09資料清理總覽CLEANING RULES

把雜亂的 OSM 變成可靠路網

抓取只是第一步——每座城市都要經過以下清理規則，才進入下游管線。

1

只留營運中路線

興建中／規劃中／停用線段一律移除，靠 lifecycle 標籤與站名括號註記雙重過濾。

2

共站合併（非「同名就合併」）

依序：stop_area 權威分組 → 同名 ≤800m 輔助 → 人工裁決 interchanges.json，union-find 取聯集。

3

環狀線收尾

真實閉環用 closeRing 補上最後一段，不斷開。

4

快慢車合併

同 network+ref 的快車/慢車/區間車合併為一條線，以慢車（站最多）的完整站表為準。

5

機廠與機場接駁電車排除

車輛段出入段線、旅客自動電車／APM／people mover 一律剔除；機場「快線」不受影響。

6

官方顏色與代碼

套用各線官方色與車站官方代碼，貼近官方地圖觀感。

7

墓碑驗證

站源殭屍（上游已刪節點）批次驗證存在性，死者拒收，避免頂點吸附誤撿。

8

對照 Wikipedia／urbanrail.net

逐城比對線數與站名，抓漏抓多抓雜訊——收斂到全部通過才算完成。

STAGE 2 · MAP ADJUST

Map Adjust ——
收成拓撲骨架與整數格

把地理路網收縮成示意圖用的骨架，保留拓撲關係、標出關鍵節點，再吸附到整數格路網，交給下一階段最佳化。

① 骨架化六步驟節點與邊分類跳站特快特殊案例② 示意格網化
13骨架化六步驟TOPOLOGY SKELETON PIPELINE

拓撲收縮，不拉直、不新增

「路線圖轉換骨架2」把「選擇路線圖」的地理路網收成示意圖佈局用的骨架——保留地理折線形狀，只做拓撲收縮與標記，與早期「會拉直」的直線骨架不同。

步驟內容
① 建圖、分節點以 route 站序相鄰兩站建無向圖，依 degree 分類紅／藍／黑
② 找真交叉逐 route 對檢查線段是否內部相交，補合成黃點節點（含同線自相交）
③ 收縮成邊從紅/藍節點沿degree=2黑點鏈直到下一節點，收成一條保留原折線的「邊」
④ 邊分類共線合併紅底／環線綠／頭尾共點藍，沿真實 edge.geom 畫襯底
⑤ 切紫點頭尾共點邊弧長 1/2 處切 1 個；環線邊 1/3、2/3 處切 2 個
⑥ 標粉紅／灰代表性轉折（曲折度 >1.25 且 Douglas–Peucker 篩選）標粉紅；過長黑點段每 5 站分隔標灰
14節點與邊分類NODE & EDGE CLASSIFICATION

七種節點角色，三種邊分類

分類依節點 degree（相異鄰站數），不是車站的 station_role——共軌段上多線一起過的站鄰居仍只有兩個，正確歸為黑點，不會整條共軌冒出一串紅。

紅 · 分歧／轉乘（degree≥3，或兩側 route 不同的 degree-2）
藍 · 真端點（degree≤1）
黑 · 直通中段站（degree=2 且兩側 route 相同）
黃 · 幾何交叉新點（合成節點，非車站）
紫 · 頭尾共點／環線切斷點
粉紅 · 曲折邊的代表性轉折
灰 · 過長黑點段的分隔點
邊分類（線底下的襯底）

共線合併 · 環線 · 頭尾共點

共線合併（≥2 相異色）標紅底；環線（自環）標綠、優先於頭尾共點；同對節點間 ≥2 條不同幾何邊標藍——都沿真實折線畫。

核心原則

線一律照原始畫法

「原始怎麼畫，骨架就怎麼畫，只是共線變一條線」——絕不用「相鄰車站直線相連」畫線，避免特快跳站段被拉直成穿越空白的長直線。

15特殊案例EXPRESS SKIP-STOP HANDLING

跳站特快：折點不一定是車站

多數折點是車站，但跳站特快段例外——特快沿本地線的真實走廊彎行卻不停中間站，那段線 feature 的幾何會有一串非車站折點（雪梨北岸線快車 Strathfield→Redfern 段有 11 個折點，只有頭尾兩個是停靠站）。

建圖只看停靠站

拓撲（degree／route 集合）一律由 routes[].stations（停靠站）建圖，不受跳站折點影響。

畫線一定用原始幾何

繪製一律用 feature 的原始幾何 path(f)，不可用「相鄰停靠站直線相連」——否則跳站段會被拉成橫越空白的長直線，骨架冒出原始沒有的線。

邊帶真實幾何 edge.geom：依該路線的停靠站切段，特快邊因此擁有整段穿過中段站的多折點曲線，供邊分類襯底貼線畫。曾試過「把跳站特快途經的本地站插入特快站序」合併共軌走廊，會捏造岔口與邊——已否決，只做純顯示層的 pass/stop 標記。

16示意格網化SCHEMATIC GRIDDING

彩色點排出藍格排名，路線吸附到整數座標

彩色點（所有非黑節點）用最密切法排出 x/y 排名，形成整數格；每條路線在彩色點切開，端點吸附、黑點在兩端之間平均放回。

① 排名吸附

每兩個相鄰不同座標間下一條分隔線，一格至多一個彩色點，撞格用 Chebyshev 環狀外擴解衝突。

② 邊界切線

每條路線在彩色點處切開，只在線內切、不跨線合併。

③ 端點吸附

每段兩端吸到整數格 (欄, 列)，與預覽藍格同一套排名。

④ 黑點均分

中段黑點沿新（拉直）小段平均放回，第 j 個站落在 j/(k+1) 處。

格網規模依城市密度而異——台北約 67×67、北京約 165×165、紐約約 270×270；欄列是等距排名，不再保留真實地理距離比例，代價交由下游最佳化處理。

Straighten ——
多準則爬山法拉直

依 Stott, Rodgers, Martínez-Ovando & Walker (2011) 的多準則最佳化演算法，把整數格佈局爬山到接近理想示意圖，再用四種方法把水平／垂直線段最大化。

17多準則爬山法MULTICRITERIA OPTIMIZATION

五項準則，逐頂點爬山

逐頂點掃描以自身為中心、半徑 R 內所有格點，取局部適應度最低且通過硬規則者移動；先算便宜的適應度，只有更優者才跑昂貴的幾何硬規則。

18硬規則、冷卻與群集移動HARD RULES & COOLING

四條硬規則否決候選，半徑逐輪冷卻

1

邊界

0 ≤ c < cols、0 ≤ r < rows，且一格一頂點。

2

相對位置

對每個鄰居保持象限，符號不得翻轉。

3

無遮蔽

候選點不落在他段上、移動後的段不吞掉其他頂點、不與不共端點的段相交。

4

邊環繞順序

與每個鄰居的入射段依角度循環序，移動前後必須循環相等（度 ≤2 恆成立）。

冷卻與增量計算

半徑 [2,1,1]，超長邊群集平移

每站每輪最多 ±2 格，逐輪縮小；移動只重算受影響的節點項與鄰接小段。超長邊（段長 >hops×L）以非超長邊做連通分量，每個真子集（≤200 頂點）整體平移，只檢查跨界硬規則。一整輪無改善即收斂。

19H/V 最大化後處理RECTILINEAR REFINEMENT

三種確定性方法，各自迭代到不動點

爬山完成後，再短距離移動彩色頂點讓水平/垂直段最多（一段是 H/V ⇔ 兩端恰有一個座標相同）。三者互相獨立、可比較，共用同一套硬規則與移動原語，都迭代到不動點（上限 20 次）。另有「端點拉直」收尾視圖：每個非白點可把一個座標吸到鄰居的欄/列，僅淨增 H/V 才動，可疊在任一條鏈之上。

直角爬山

八方向準則換方向

|sin(4θ)| 換成 |sin(2θ)|（45° 變最貴），權重 ×3、半徑 [2,1,1] 再爬一次。

軸對齊

方向指派＋座標指派

逐軸把貼近該軸（45° 不算）、垂距 ≤2·maxShift 的段用 union-find 併群，每群吸到成員座標中位數（L1 最小移動）。

整數規劃

逐軸 0-1 精確解

每個可對齊端點一個 offset 變數 ∈[-k,k]，最大化對齊段數（位移當 tie-break）；配對圖分連通元件，生成樹 DP 求解，環用小 feedback set 枚舉。

RWD Maps ——
版面路網畫線

把 Straighten 的縮減網格重繪成嚴格 H/V/45° 折線；八方向約束以「畫面像素」為準，版面一變就在新像素座標重新求解，圖隨板面變形。

同軌路線合併候選折線絕不交叉的五層消解軟目標收尾流量權重LLM 調整
22同軌路線合併PARALLEL SEG MERGING

重疊就一條，不錯開成平行線

使用者裁決：RWD 要跟匯入的 network 一樣，同軌路線重疊就畫一條、不錯開。判準一律看匯入的真實幾何，不管官方圖語意。

(A) 同端點

共用兩個 cut 端點

快車直達＋普通車等共端點路線，站數最多的當代表、其餘 routes 併入，其 seg 丟棄。

(B) 幾何重疊

快車跨過中間轉乘節點

快車邊整條 edge.geom 都貼在另一組邊的軌道上（偏差 <10m），即使跳過中間 junction，仍併入這條鏈。

涵蓋性防呆（絕不可省）：只有當被併掉的邊每一站都已在保留路徑上才併——否則被丟掉的獨有站會孤立。一條獨立線（如 Sydney Metro）可以和市郊鐵路共軌但有自己的車站，這種一律不併。實測全 224 城 0 孤立站。

23候選折線與轉折優先序CANDIDATE ROUTING

直線 > 單折 > 雙折，轉折數絕對優先

每段依轉折數由少到多產生候選折線，所有選擇點都取「第一個零衝突候選」。方向判準：Δy≈0→H、Δx≈0→V、|Δx|≈|Δy|→45°。

直線

45°轉45° 禁止：折線不得有兩條相鄰腿皆為斜線（不同族），因此 H/V 線沒有合法單折繞行（唯一單折是被禁的 45-45 三角），只剩雙折平行偏移。

24絕不交叉的五層消解CONFLICT ESCALATION

候選都衝突時，依序升級五個層級

「路線交叉絕不允許」是最高優先硬規則；衝突判定含共線重疊/平行貼線、線段相交（端點對端點除外）、壓過節點三種。

1

衝突消減重掃

全部繞線完後重訪每條仍有衝突的線，以完整資訊重選最少衝突候選，至多 4 輪。

2

A* 格點佈線

半格 lattice 上找只穿過自由空間的路徑；正方虛擬格時斜向步進也合法（8 方向）。

3

Rip-up & Reroute

PCB 佈線手法：被封死（Jordan 曲線圍死）時，拆掉最常撞到的牆線，先畫被關的段、再讓牆線改繞。

4

Restart-with-priority

仍失敗的段下一輪全圖重排時最先繞線（牆還沒築起來），原牆線改繞它們；至多 6 輪。

5

窄縫救援

最後仍無路的段以 60% 貼線距離再 A* 一次，交叉依然絕禁。

實測：台北／倫敦／北京 0 殘留衝突；全 224 城 0 孤立站、0 非法角度兜底；紐約曼哈頓密網 5/451 段列為已知極限。

25軟目標與收尾迭代SOFT GOALS

同名路線盡量不轉折，L 形能改 45° 就改

衝突消解後的微調，排在折數絕對優先序之後——不會為此增加任何段的折數。

順接軟調整

續接角評分

同轉折數的候選間互換，分數＝與共站同路線折線的續接角——直線續接(180°) 最佳，依序 135°、90°、45°。

L→45（最後一步）

能改就必須改

純水平接垂直的單折 L 形，若改成「45°＋直線」不與任何線重疊，就必須改——排在收尾迭代的最末，結果不再被其他 pass 改動。

收尾迭代跑兩輪：〔順接軟調整 → 降折到定點 → L→45（最後）〕，每步挪動走廊後回頭重撿。

26流量權重與響應式概括化WEIGHT-DRIVEN GENERALIZATION

依流量放大主幹、隱藏次要站

版面簡化不改拓撲——只把權重轉成非均勻的欄寬列高，在新像素座標重跑 RWD 繪線。weight 粒度＝相鄰兩站的 link，不是段的兩端點。

非均勻欄列

取 max 不取 sum

欄寬列高由 link weight 取最忙那條決定；權重 0 的欄列不消失（保留 ≈0.25× 均勻寬的下限），外框固定。

自動隱藏白點

全域 cutoff 一致

逐段算「均分後站距 ≥ 最小站距 pt 所需的 weight 差門檻」，取 max 為全域 cutoff T；任何白點左右 link 差 ≤T 就隱藏，剩餘站沿弧長重新分配。

權重改變不瞬跳——~700ms 內內插起點到目標的格線位置，每幀用內插後座標重算並跑快速版 buildRwdMap，最後一幀走完整品質（settle 到無交叉版面）。

27LLM 調整工作流程NATURAL-LANGUAGE GRID WEIGHTS

一句話，讓 AI 推理每欄每列的顯示權重

第二種「權重＝大小」來源：使用者說「把市中心那幾欄拉開」，語言模型逐欄／逐列推理顯示權重——不搬任何站的整數格座標，只改顯示比例，不做任何幾何求解。

{ "model": "Fable 5", "colW": [1, 1, 4.5, 3, 1, 0.8], "rowW": [0.9, 1, 3.8, 1.2], "note": "市中心兩欄放大 4.5x／3x，緊鄰一欄緩衝", "userPrompt": "把市中心那幾欄拉開" }

規則：必須給全部區間權重、只能回區間 weight（不可回單點座標）；核心目標區間縮放要明顯（3–5 倍，至少一組 ≥3）；漸近而非單格尖峰——核心最大、緊鄰緩衝、外層回到 1。系統負責正規化外框與重畫，模型只決定意圖與強度。
