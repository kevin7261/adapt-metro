Adapt-Metro · SYSTEM INTRODUCTION

從地鐵路網到
響應式示意圖

一個純前端、零求解器依賴的四階段管線 —— Metro Maps → Map Adjust → Straighten → RWD Maps。 234 座城市的真實路網、多準則爬山最佳化＋三步鏈循環壓縮（端點移動→直線縮減→網格合併）、嚴格方向約束折線（4／8／16 方向可選，預設 H/V/45° 八向），全部在瀏覽器裡完成，隨版面即時重排，而且每一個單一移動都能逐步重播。這份投影片逐一拆解每個階段實際在做的事。

INPUT

真實地理路網

OpenStreetMap / Overpass 抓取的地鐵與輕軌路線、車站座標——保留真實地理形狀，經清理與逐城驗證。

OUTPUT

響應式示意圖

只用嚴格方向約束的折線（預設 H/V/45°，可選 4 或 16 方向），版面一變就在新像素座標重新求解，絕不交叉。

四個圖層群組，串成一條產線

每個階段都是介面「Layers」面板裡的一個群組，可以逐步檢視、比較、回溯——不是黑盒子一次生成。後面每一節會逐項拆解這四個階段內部的機制。

版面畫線

4／8／16 方向折線繪製（預設 H/V/45°），依版面、裝置模擬與流量權重即時重排。

純函式、確定性、無亂數——同一份輸入永遠得到同一份輸出，唯一的例外是選用性的 LLM 對齊／互動／評價／比較（離線預算、存檔、網頁只載入）。

02示意圖的定義WHAT IS A SCHEMATIC MAP

扭曲真實地理，只保留拓撲關係

捷運路線圖是一種 schematic map：犧牲精確地理位置，換取每條路線都走水平／垂直／45°（八向）的可讀性。整條管線做的事，就是把左邊的真實路網，逐步變成右邊的樣子。

04城市判定與合併CITY RESOLUTION

「同一城市＝同一系統」的三層防護

SKILLmetro-osm-fetch城市規則metro-cities

OSM 車站常無 network 標籤，城市名稱解析容易撞名（如 S-Bahn Frankfurt 的 bahn token 撞上 Berlin U-Bahn）——三層護欄避免抓錯城市。

1 · token 黑名單

泛用交通詞（bahn／ubahn／sbahn／lrt／tram／verkehrsverbund…）不得參與城市名稱比對。

2 · rebucket 距離護欄

member network 解析出的目標城市，須離群組重心 ≤250 km 才准搬家；全國性營運商（如 DB）重心無意義。

3 · 線級距離護欄

解析出的 wiki 城市離該線重心 >250 km ⇒ 改用線自身座標或原始反查城市，自成獨立系統檔。

都會圈合併 CITY_MERGE

鄰接行政市併成一城一檔（如桃園→台北）；被併城市在驗證時以 covered_by 標註通過。

LRT 範圍規則

附掛輕軌的城市（台北／高雄／新加坡）附加收錄；整系統即為 LRT tagging 者（Vancouver SkyTrain）全收；純附掛輕軌原則剔除，但覆蓋率 <0.6 時例外保留。

06資料格式 · 每城一檔DATA FORMAT · GEOJSON

一份 FeatureCollection：系統中繼資料＋兩種 feature

每座城市一個 GeoJSON（systems/{洲}/{國}/{洲2碼}-{IOC3碼}-{城}.geojson，如 as-twn-taipei.geojson）：頂層掛 metro_system 系統中繼資料，features 只有兩種——車站 Point 與 路段 MultiLineString。

{ "type": "FeatureCollection", "metro_system": { "continent": "asia", "country": "Taiwan", "city": "Taipei", "operator": "臺北大眾捷運股份有限公司", "official_website": "…", "wikidata": "Q…", "line_count": 18, "segment_count": 20, "station_count": 197, "audit": { …逐城驗證結果 } }, "features": [ { …Point 車站 }, { …MultiLineString 路段 } ] }
幾何語意 1

線永遠壓在站點上：路段幾何＝共站合併後的車站點依站序連線，每個折點、端點都是車站（跳站特快段例外，見 Stage 2）。

幾何語意 2

重疊路段只畫一條：相鄰車站對被多線共用時只輸出一個 feature，routes list 記行經路線——一條 route 的完整路徑＝所有含它的路段之聯集。

幾何語意 3

快車一律不抓：只跳站的快車（站點是慢車子集）不另成線——各停已涵蓋全部車站；像紐約 J/Z 交錯停站或獨立編號快線（香港 AEL）自然保留，跳過的站沿慢車走廊標 pass 畫共線。

09資料清理總覽CLEANING RULES

把雜亂的 OSM 變成可靠路網

SKILLmetro-osm-fetchmetro-audit

抓取只是第一步——每座城市都要經過以下清理規則，才進入下游管線。

1

只留營運中路線

興建中／規劃中／停用線段一律移除，靠 lifecycle 標籤與站名括號註記雙重過濾。

2

共站合併（非「同名就合併」）

依序：stop_area 權威分組 → 同名 ≤800m 輔助 → 人工裁決 interchanges.json，union-find 取聯集。

3

環狀線收尾

真實閉環用 closeRing 補上最後一段，不斷開。

4

快車一律不抓

只跳站的快車（子集）捨棄，各停站表為準；交錯停站（NYC J/Z）或獨立編號快線（AEL）自然保留、跳站標 pass 畫共線。

5

機廠與機場接駁電車排除

車輛段出入段線、旅客自動電車／APM／people mover 一律剔除；機場「快線」不受影響。

6

官方顏色・站碼・在地語言

各線官方色、車站官方站碼（並依碼正規化站序，A1 在前）、站名/線名一律在地語言（台繁/中簡/日日）＋英文副標。

7

墓碑驗證

站源殭屍（上游已刪節點）批次驗證存在性，死者拒收，避免頂點吸附誤撿。

8

對照 Wikipedia／urbanrail.net

逐城比對線數與站名，抓漏抓多抓雜訊——收斂到全部通過才算完成。

13骨架化六步驟TOPOLOGY SKELETON PIPELINE

拓撲收縮，不拉直、不新增

SKILLroute-skeleton-connect實作src/stores/skeleton.js

「路線圖轉換骨架2」把「選擇路線圖」的地理路網收成示意圖佈局用的骨架——保留地理折線形狀，只做拓撲收縮與標記，與早期「會拉直」的直線骨架不同。

步驟內容
① 建圖、分節點以 route 站序相鄰兩站建無向圖，依 degree 分類紅／藍／黑
② 找真交叉逐 route 對檢查線段是否內部相交，補合成黃點節點（含同線自相交）
③ 收縮成邊從紅/藍節點沿degree=2黑點鏈直到下一節點，收成一條保留原折線的「邊」
④ 邊分類共線合併紅底／環線綠／頭尾共點藍，沿真實 edge.geom 畫襯底
⑤ 切紫點頭尾共點邊弧長 1/2 處切 1 個；環線邊 1/3、2/3 處切 2 個
⑥ 標粉紅／灰代表性轉折（曲折度 >1.25 且 Douglas–Peucker 篩選）標粉紅；過長黑點段每 5 站分隔標灰
14節點與邊分類NODE & EDGE CLASSIFICATION

七種節點角色，三種邊分類

SKILLroute-skeleton-connect

分類依節點 degree（相異鄰站數），不是車站的 station_role——共軌段上多線一起過的站鄰居仍只有兩個，正確歸為黑點，不會整條共軌冒出一串紅。

紅 · 分歧／轉乘（degree≥3，或兩側 route 不同的 degree-2）
藍 · 真端點（degree≤1）
黑 · 直通中段站（degree=2 且兩側 route 相同）
黃 · 幾何交叉新點（合成節點，非車站）
紫 · 頭尾共點／環線切斷點
粉紅 · 曲折邊的代表性轉折
灰 · 過長黑點段的分隔點
邊分類（線底下的襯底）

共線合併 · 環線 · 頭尾共點

共線合併（≥2 相異色）標紅底；環線（自環）標綠、優先於頭尾共點；同對節點間 ≥2 條不同幾何邊標藍——都沿真實折線畫。

核心原則

線一律照原始畫法

「原始怎麼畫，骨架就怎麼畫，只是共線變一條線」——絕不用「相鄰車站直線相連」畫線，避免特快跳站段被拉直成穿越空白的長直線。

15特殊案例EXPRESS SKIP-STOP HANDLING

跳站特快：折點不一定是車站

SKILLroute-skeleton-connect

多數折點是車站，但跳站特快段例外——特快沿本地線的真實走廊彎行卻不停中間站，那段線 feature 的幾何會有一串非車站折點（雪梨北岸線快車 Strathfield→Redfern 段有 11 個折點，只有頭尾兩個是停靠站）。

建圖只看停靠站

拓撲（degree／route 集合）一律由 routes[].stations（停靠站）建圖，不受跳站折點影響。

畫線一定用原始幾何

繪製一律用 feature 的原始幾何 path(f)，不可用「相鄰停靠站直線相連」——否則跳站段會被拉成橫越空白的長直線，骨架冒出原始沒有的線。

邊帶真實幾何 edge.geom：依該路線的停靠站切段，特快邊因此擁有整段穿過中段站的多折點曲線，供邊分類襯底貼線畫。曾試過「把跳站特快途經的本地站插入特快站序」合併共軌走廊，會捏造岔口與邊——已否決，只做純顯示層的 pass/stop 標記。

23movewise 三步鏈MOVEWISE COMPACTION CHAIN

每一個移動後，立即縮減網格

SKILLroute-movewise-loop跨距上限route-span-cap

九條鏈的結果再各走一條三步鏈：端點移動 → 直線縮減 → 網格合併。「縮減網格」不是獨立步驟——每一個單一移動完成後就把整排/整欄沒有彩色點的欄列移除（movewise），網格隨時緻密、尺寸逐移動縮小。

共同守則 1

一次只移一格

三個演算法一致；遠距離的對齊與合併靠「移一格→壓縮→再移一格」逐步完成。

共同守則 2

跨距上限

任何移動不得讓受影響段的兩個顏色點橫跨超過 n 格（Chebyshev；Straighten 工具列輸入、預設 3、改數字即重算——只約束爬山 movewise，RWD 畫線不用）。舊長段只准縮短。

共同守則 3

拓撲鐵律

所有移動走同一套硬規則：不壓點、不新增交叉、不產生路線重疊、象限與邊環繞序不變——怎麼壓，拓撲都不變。

共同守則 4

單調收斂

每種移動各守一個嚴格單調量（H/V 數、網格欄列數…），不用能量函數也保證終止。

24三步鏈的三個演算法THREE MOVE KINDS

點移一格、線移一格、邊界兩兩合併

SKILL1route-endpoint-move2route-line-compact3route-grid-merge
1 · 端點移動

每個非白點移 1 格

採納條件三擇一：入射段 H/V 淨增；H/V 不變但「直線變長且斜線變短」（把斜線一格一格吃掉）；藍色端點「線變短」（往鄰居收）。bendsPaid 護欄：既有直段只有同路線同步被拉直才准折彎。

2 · 直線縮減

整條直線垂直平移 ±1

「直線」＝同軸共線段跨相交點串接的連通元件（轉乘/交叉點被直穿就整條一起動）。水平線只上下移、垂直線只左右移；必須嚴格減少佔用欄列數、且全網 H/V 段數不減。

3 · 網格合併

row/col 兩兩合併

合併 r|r+1＝「row>r 的所有點整體上移一格」（半平面剛體平移、自帶壓縮），col 同理。合法性走與前兩者相同的硬規則；附帶保證跨距只縮不增、H/V 只增不減。

收斂單調量分別是 (−H/V 數, 斜線長, 總長) 字典序、網格欄列數嚴格遞減、每次合併欄列數 −1——全部有界，保證終止。

25循環與逐步驗證ROUND-ROBIN & STEP-THROUGH

三個演算法各掃一遍輪替；每一步都能重播

SKILL循環route-movewise-loop逐步route-step-verify
循環

掃一遍就換手，全靜止才停

每輪＝端點移動把整個 network 掃一遍（每個元素本輪最多動一次）→ 直線縮減掃一遍 → 網格合併掃一遍 → 回到端點移動；某一輪三個演算法都沒有任何改動才停止。RWD Maps 的底圖就是這個循環的收斂結果。

逐步驗證

下一步／下一小步／上一步

「下一步」＝目前演算法掃一遍、掃完換下一個；「下一小步」＝這一遍中的下一個單一移動，圖上以前後比對標示（舊位置虛線空心圈、虛線軌跡、新位置橘色實圈、訊息附座標）；「上一步／上一小步」快照復原；三階段 chips 顯示目前工作。

逐步驗證全用大步跑到收斂的結果，與循環 tab 完全一致——逐步不是示意動畫，就是同一個演算法被一步一步執行。

28候選折線與轉折優先序CANDIDATE ROUTING

直線 > 單折 > 雙折 > 多折，轉折數絕對優先

SKILLroute-rwd-draw

每段依轉折數由少到多產生候選折線：直線、單折（45° 或 L）、雙折兩家族（45–H–45／H–45–H，分點 1/2·1/4·3/4）、三折混合（axis–45–axis–45）、45° 與軸腿任意次交替的階梯（折數硬頂 ≤20）。方向判準以像素為準：Δy≈0→H、Δx≈0→V、|Δx|≈|Δy|→45°。

直線
0 轉折・規則 0 鐵律：可直線且零衝突就鎖定
單轉折
L 形或 45°收尾
雙轉折
45–H–45／H–45–H 兩家族
交替階梯
45° 與軸腿任意次交替
兜底
全衝突才用非法角度（實測全城 0）
45° 轉 45° 禁止

折線不得有兩條相鄰腿皆為斜線——斜腿一律被軸向直線隔開；H/V 線因此沒有合法單折繞行（唯一單折是被禁的 45-45 三角），只剩單折大繞行與雙折平行偏移。

直角樓梯禁止

對角走向的段要用 45° 斜線，不畫一階一階的 H-V-H 樓梯（vh-vh-vh）；A* 救援在非正方版面只能走直角、產生的樓梯由收尾 destairPass 換回 45° 折線。

29方向級與候選選擇序DIRECTION LEVELS & PICKBEST

4／8／16 方向：能用低一級就不用高一級

SKILLroute-rwd-draw

工具列「方向」下拉三選一（預設 8），三個方向級同一套判斷邏輯。方向級優先是 tie-break 不是硬閘：衝突數（含近距貼線）排在方向級之前——零衝突的 22.5° 贏過有重疊或貼線的 45°；兩者都乾淨時 45° 一定優先。

4 方向（只 H/V）

不產任何 45° 候選；對角段用同一組候選骨架、每條 45° 腿展開成 H/V，另直接生成真 Z 字（V–H–V／H–V–H，分點 50/25/75）；A* 純直角步進。實測全城 45°＝0。

8 方向（預設）

＋45°。本節所述的完整候選折線規則。

16 方向（＋22.5°/67.5°）

加 22.5° 族候選：單折、雙折（斜–軸–斜／軸–斜–軸）、45＋22.5 混合家族（同一條路可以 45 與 22.5 組合）；斜腿仍一律被軸向腿隔開。收尾 deskewPass 把走廊空出後的 22.5 段升回 45 級——折數可增（方向級優先於折數）。

候選選擇序（pickBest，八鍵字典序）

① 衝突數最少 → ② 近距貼線最短（視覺上的貼線也是「重疊」）→ ③ 45 優於 22.5° → ④ 折數最少 → ⑤ 硬共線重疊最短 → ⑥ 分點 50%→25%→75% → ⑦ 軟共線最短 → ⑧ 路徑總長最短（線越短越好，最弱一階）。

30絕不交叉的多層消解CONFLICT ESCALATION

候選都衝突時，依序升級七個層級

SKILLroute-rwd-draw

「路線交叉絕不允許」是最高優先硬規則。衝突判定：共線重疊/平行貼線（累積 ≤25% 弧長硬上限）、線段相交（端點對端點除外，同一對線最多交叉一次）、壓過節點直接淘汰、節點環狀順序不可變（rotation system——同節點任兩條線不得互換位置）。

1

衝突消減重掃 ＋ 成對線聯合重算

重訪每條仍有衝突的線重選候選（至多 4 輪）；仍彼此交錯的一對直接偵測真交叉入場，兩條同時拆掉、聯合搜 bounded-bend 候選組合（方向級優先於總折數），深輪再升級成對 rip——「這對線到底有沒有不交叉畫法」的實質驗證，鎖定直線也解鎖讓路。

2

A* 格點佈線（大環繞禁止）

半格 lattice 上找只穿過自由空間的路徑；限制在 S–T 走廊窗內、路徑成本設上限——寧可留下局部殘留衝突，也不畫繞整張圖的環繞線。正方格時斜步合法（8 方向）。

3

Rip-up & Reroute

PCB 佈線手法：被封死（Jordan 曲線圍死）時，拆掉最常撞到的牆線，先畫被關的段、再讓牆線改繞。

4

Restart-with-priority

仍失敗的段下一輪全圖重排時最先繞線（牆還沒築起來），原牆線改繞它們；至多 6 輪。

5

窄縫救援 → 共線救援 → forced

以 60% 貼線距離再 A* 一次；再不行放寬「共線/貼線」約束（交叉與壓點依然絕禁）——共線是可接受的次佳解。全部失敗才標 forced（琥珀殘留衝突＝深度 Jordan 封閉的極限案例）。

實測全 234 城：0 孤立站、0 非法角度兜底；成對聯合重算上線後殘留交叉對 139→112、forced 208→132（多為紐約/東京 JR/巴黎等密網角落）。

31軟目標與收尾迭代SOFT GOALS & FINISHING PASSES

同名路線盡量不轉折，L 形能改 45° 就改

SKILLroute-rwd-draw

衝突消解後的微調，排在折數絕對優先序之後——不會為此增加共線或交叉。收尾迭代跑兩輪：〔A* 樓梯轉 45（destairPass）→ 22.5 升回 45（deskewPass）→ 壓短共線（移分點）→ 順接軟調整 → 降折到定點 → L→45（最後）〕，每步挪動走廊後回頭重撿。

順接軟調整

續接角評分

同轉折數的候選間互換，分數＝與共站同路線折線的續接角——直線續接(180°) 最佳，依序 135°、90°、45°。

降折到定點＋協商式拉直

能少折就少折，能直線就直線

降折不得讓軟共線變長；更直的畫法只被 1–2 條線擋住時，拆掉那些線、採用直畫法、擋路線改繞，全部乾淨才提交。變成直線時不設折數預算——已經彎的鄰線要多彎幾折都得讓。

destairPass ＋ deskewPass

樓梯換 45、22.5 升回 45

A* 救援在非正方版面產生的直角樓梯子段用同一套候選重畫成 45° 折線；16 方向下衝突消解後走廊空出的 22.5 段升回乾淨的 45 級（折數可增，方向級優先於折數）。

L→45（最後一步）

能改就必須改

純水平接垂直的單折 L 形，若改成「45°＋直線」不與任何線重疊就必須改——優先選同路線鄰段同方向 45° 的那端，結果不再被其他 pass 改動。

33流量權重與響應式概括化WEIGHT-DRIVEN GENERALIZATION

依流量放大主幹、隱藏次要站

SKILLroute-rwd-draw權重→尺寸src/stores/rwdWeight.js

版面簡化不改拓撲——只把權重轉成非均勻的欄寬列高，在新像素座標重跑 RWD 繪線。weight 粒度＝相鄰兩站的 link，不是段的兩端點。

非均勻欄列

取 max 不取 sum

欄寬列高由 link weight 取最忙那條決定；權重 0 的欄列不消失（保留 ≈0.25× 均勻寬的下限），外框固定。

自動隱藏白點

全域 cutoff 一致

逐段算「均分後站距 ≥ 最小站距 pt 所需的 weight 差門檻」，取 max 為全域 cutoff T；任何白點左右 link 差 ≤T 就隱藏，剩餘站沿弧長重新分配。

權重改變不瞬跳——~700ms 內內插起點到目標的格線位置，每幀用內插後座標重算並跑快速版 buildRwdMap，最後一幀走完整品質（settle 到無交叉版面）。

35LLM 評價與 LLM 比較LLM AS CRITIC & JUDGE

LLM 評價（route-llm-eval）

一次比較原始與旋轉變體的論文①〜⑧八條鏈與可用的 LLM 對齊（最多 18 個），依方正/直線比例、轉折數、平衡、硬失敗（forced/fallback）評分，選出 winner、winnerOrig、winnerRot 並逐一列優缺點。純評審——不動任何座標。

第九條直線鏈是 LLM 對齊（不對應論文、由模型當最佳化器）。另有 Boeing 2019《Urban Spatial Order: Street Network Orientation, Configuration, and Entropy》——非上表編號文獻，但有直接程式對應：src/stores/orientation.js 的長度加權方向熵／φ 計算（Stage 2 示意格網化 rotated 變體的主軸角偵測）直接沿用該文公式。

41三步鏈的定位POSITIONING

犧牲最佳性，換取秒級速度與逐步可視化

固定拓撲下的正交示意化＋面積最小化是經典 NP-hard 問題——TSM 框架的 orthogonal compaction、VLSI 1-D compaction、Misue et al. 的 orthogonal ordering 保持都是零件級近親。

本系統的做法

Local search，非求解器

每種移動各守一個嚴格單調不變式，且每個移動後立即全域壓縮（movewise）——用局部搜尋逼近最佳解，不依賴 ILP／SAT 求解器。

換到的東西

秒級全量速度＋逐步可視化

234 城全量約 5 秒收斂；犧牲最佳性換取速度，且每一個單一移動都可以在逐步驗證裡重播、比對前後——這是求解器路線（ILP/SAT）給不了的。
