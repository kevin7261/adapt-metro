# 示意圖佈局（示意格網化）(route-skeleton-grid)

把 [[route-skeleton-connect]] 骨架（原始骨架化 或 旋轉骨架化）收成**整數格路網**：彩色點用藍色
分隔網格做**排名吸附**，每條路線在彩色邊界切開，黑點沿新邊平均放回。此 skill 是這個轉換的
**唯一權威依據**。

> **不重做骨架拓撲、不跨路線合併**；每條骨架邊各自處理。純幾何、零網路零 token。
> 實作：`src/stores/schematicGrid.js` 的 `buildSchematicGrid(skeleton, posById, extent)`；
> UI 在 `D3Tab.vue`（Map Adjust）的兩個 tab：**格網化前**（畫藍格預覽、座標未改）與
> **格網化後**（吸附後的整數格路網）；**兩者都在上面畫藍格**。

## 整體流程

```
骨架（線 + 站，來自骨架化）
  → ① 蒐集彩色點，用 x/y 排名建藍色分隔網格（欄、列）
  → ② 每條線在彩色邊界切開（同一條線內切、不跨線）
  → ③ 各小段端點吸附到整數格（欄, 列）
  → ④ 黑點沿新（拉直）小段平均放回
```

## 點的分類（沿用骨架的 `stationClass`）

**彩色點＝所有非黑的節點**：紅（分歧/轉乘）、藍（端點）、黃（幾何交叉）、紫（切斷）、
粉紅（轉折）、灰（分隔）（＋棕，若有）。這些都是**邊界**——排名與切線都用它們。
**黑點（直通站）不切格、不佔欄列排名**。

## ① 排名吸附（藍色分隔網格）

- **粗切＋局部加線（兩段式，保持格網稀疏）**：
  1. 先**粗**分帶：鏈式聚類 `TOL=10px`（間隙 >10px 的中點下一條分隔線）——格數少、視覺乾淨。
  2. 再**只在違規處加線**：某格若有 ≥2 個彩色點，找該格內**最大間隙**（x 或 y 取大者）的中點
     **加一條分隔線**，重驗到每格 ≤1 為止。格網最終＝「粗格＋幾條必要的線」，
     不是一點一帶的密格（曾試過全細切 → 53×48 過密，被使用者否決；實測本法
     Taipei 22×21、Beijing 32×25、NYC 43×42，均 0 碰撞）。
- 唯一例外：兩彩色點**兩軸間隙都 <0.5px**（圖上疊在同一點、無縫可切）→ 預覽維持重疊，
  格網化後由撞格 bump（最近空格）分開，POST 保證 0 碰撞。
- 每個彩色點得到 `(欄, 列)` 整數座標（rank = 位置前的分隔線數）。
- **硬性：一格至多一個彩色點（白/黑除外）**。若兩彩色點被排到同一格 → **撞格解衝突**：
  將後者外擴到**最近的空格**（Chebyshev 環狀搜尋，位移最小），保證每格 ≤1 彩色點。
- **藍線是「分隔線」——畫在格與格之間，絕不穿過點**；每個彩色點吸附到其格子的**中央**。
  （早期版把點放在格線交點上是錯的：視覺上會出現一格多點/點壓線。）
- **代價**：欄列是**等距排名**，不再保留真實地理距離比例（之後若有落線/交叉交給下游處理）。
- 藍格畫在**最底層**：**前**＝相鄰彩色點群中點畫分隔線（預覽，每個 rank 帶各自成一格）；
  **後**＝解衝突後的等寬格界（cols+1／rows+1 條線），格座標標在**格子中央**（下方欄、左方列）。

## ② 每條線在彩色邊界切開

對骨架每條路線折線：起點、終點一定是切點；中段頂點若是彩色點也是切點。
相鄰切點間成一段「小段」。**只在同一條線內切，不跨線合併。**

## ③④ 端點吸附 + 黑點均分

每一小段：兩端（彩色點）吸到整數格 `(欄,列)`；中間黑點沿「兩端整數格連線」**平均放回**
（第 j 個黑點在 j/(k+1) 處）。端點吸附與預覽藍格**同一套**排名，前後格線意義一致。

## 硬性原則

1. **黑點不切格、不佔欄列排名。**
2. **紅／藍／黃／紫／粉紅／灰（及棕）都是邊界**——線在這些點切開。
3. **每條 way 各自處理**，不跨 way 併邊改拓撲。
4. **起訖端點吸附與預覽藍格同一套排名**，開始執行前後格線意義一致。
5. **路線中間不該無故冒紅／藍**：那是骨架階段已分好的；⑨ 只平移到整數格並重放黑點。

## 一句話

彩色點排出藍格排名 → 路線在彩色點切開 → 端點吸到整數欄列並拉直 → 黑點在兩端之間平均插回。

## 實作契約

- `buildSchematicGrid(skeleton, posById, extent)` 為**純函式**：`skeleton` 為 connect 骨架、
  `posById` 為 `Map<id,[x,y]>`（已投影的螢幕座標，含旋轉）、`extent=[x0,y0,x1,y1]`。回傳
  `{ posAfter: Map<id,[x,y]>, blueBefore:{xs,ys}, blueAfter:{xs,ys}, cols, rows }`。
- 排名用彩色點（`stationClass !== 'black'`）的 x/y 分群；黑點只由小段內插得到 posAfter。
- **撞格解衝突已實作**（`nearestFree`，Chebyshev 環狀外擴）——保證每格 ≤1 彩色點。
- 藍格畫在**最底層**（在線與站點之下）。落線/交叉矯正仍列為下游。改時同步本檔。

## 修改此轉換時

排名/切線/吸附規則變動，**同步更新本 SKILL.md 與 `src/stores/schematicGrid.js`**。
