# 路線圖轉換骨架2（connect 骨架）(route-skeleton-connect)

把「選擇路線圖」的地理路網，收成示意圖佈局用的 **connect 骨架**：共線併成一條邊、真交叉補點、
紅／藍錨點保留、中段站變黑；並在骨架邊上標出**粉紅（轉折）**與**灰（過長黑點段分隔）**。
此 skill 是這個轉換的**唯一權威依據**，實作要與此處一致。

> **不拉直**：保留地理折線形狀，只做**拓撲收縮與標記**（與早期「直線骨架」不同——那個會拉直）。
> 純幾何/圖論轉換、**零網路、零 token**：只讀已載入的 GeoJSON（[[metro-osm-fetch]] 產出），
> 不改原始資料，輸出節點分類／邊分類／標記點供繪製；可即時開關（骨架化 ⇄ 還原）。
> 實作：`src/stores/skeleton.js` 的 `buildConnectSkeleton(geojson)`；UI 在 `D3Tab.vue` 的
> 「骨架化」按鈕（Map Adjust 群組的 D3.js 視圖）。

## 資料前提

本資料的線幾何＝「共站合併後的車站點依站序連線」（[[metro-osm-fetch]]，不抓真實軌道線形）——
**每個折點都是車站**。故圖論的「節點＝車站」、「邊＝route 站序中相鄰的兩站」，共線（重疊路段）
已由資料端去重成 `route_count≥2` 的路段。每條 route 的站序讀 `routes[].stations`。

## 節點分類（依圖的 degree，不是 station_role）

以「route 站序相鄰兩站」建無向圖；`degree` = 相異鄰站數；每條邊記「經過的 route 集合」。

| 顏色 | 條件 |
|--|--|
| 紅 | 分歧／轉乘：`degree ≥ 3`，**或** `degree = 2` 但兩側邊的 route 集合不同（route 換線點） |
| 藍 | 真端點：`degree ≤ 1` |
| 黑 | 直通中段站：`degree = 2` 且兩側 route 集合相同 |

為什麼這樣分（**不用 station_role**）：
- 共軌段上「多線一起過、前後站相同」的站，鄰居仍只有兩個 → **黑**，不會整條共軌冒一串紅。
- 一條線被切成多段、段接點若 `degree=2` 且同 route → **黑**，串成連續骨頭，不被當藍端點打斷。

紅／藍是**強制骨架節點**；收縮時**絕不穿過**。

## 演算法

```
① 建圖、分節點（紅/藍/黑）
② 找真交叉 → 黃點
③ 收縮 degree-2 黑點鏈成「邊」（保留原折線形狀 + 經過的 route）
④ 邊分類（共線紅底 / 環線綠 / 頭尾共點藍）
⑤ 切紫點（藍 1/2、綠 1/3·2/3；共線不切）
⑥ 標粉紅（轉折）與灰（過長黑點段分隔）
```

### ② 找真交叉，補黃點
對每兩條**不同** route，檢查所有線段對是否在**內部**相交（非端點對端點的共點）。
交點原本折線上就有頂點 → 不算新交叉；原本沒有 → **黃點（幾何交叉節點）**，插進折線。
（metro 資料多數交叉本身即為轉乘站，黃點少見；v1 先略、標為 v2，見文末。）

### ③ 收縮成邊
從每個紅/藍節點沿每個鄰邊，一路穿過 `degree=2` 黑點直到下一個節點——這串站＝一條**邊**。
邊保留**原折線（地理形狀不變）**與其**經過的 route 集合**；中段黑點仍在原位置畫，不消失。
純環（整條 route 皆黑、無紅/藍）自成一條首尾同節點的**自環邊**。

### ④ 邊分類（顯示底色）
**線一律照原本畫法**（單線＝route 原色、重疊＝各 route 交錯彩色虛線 `dasharray [0,i·D,D,(n−1−i)·D]`、
n 依 MAX_OVERLAP=6 夾住）——與 metro map 完全相同。邊分類的顏色**不改線色**，而是畫成
**線底下的 highlight（襯底）**：較寬、半透明的底色線。

| 類別 | 條件 | 襯底 highlight |
|--|--|--|
| 共線合併 | 邊上 route ≥2 | **紅**底；**不切紫點** |
| 環線 | 邊首尾同節點（自環） | **綠**底；**優先於**頭尾共點 |
| 頭尾共點 | 同一對節點 A–B 間有 ≥2 條不同幾何邊（平行多重邊），且非環線 | **藍**底 |
| 一般 | 其餘 | 無襯底 |

### ⑤ 紫點切斷
| 邊類型 | 切斷位置 |
|--|--|
| 共線合併 | 不切 |
| 頭尾共點（藍） | 弧長約 **1/2** 處切 1 個 |
| 環線（綠） | 弧長約 **1/3、2/3** 處切 2 個 |
取點：取最接近該弧長比例的內部黑點升為**紫**節點。

### ⑥ 粉紅（轉折）與灰（過長段分隔）
**粉紅＝代表性轉折點**，從骨架邊上的中段黑點挑出，只看「這條邊夠不夠彎」＋「哪幾個彎夠代表性」。
逐邊兩道關卡（與路線大小無關，只看相對彎曲）：

1. **曲折度關卡**：曲折度 = **弧長 ÷ 兩端直線距離（弦長）**。≈1 幾乎直線、越大越彎；頭尾幾乎
   重合（環）→ 弦長≈0 視為**極曲折**。**曲折度 ≤ 1.25（弧只比直線長約 25% 內）→ 這條邊不挑粉紅。**
2. **夠彎 → Douglas–Peucker 找代表性轉折**：容差是**比例不是公尺**——子段頭尾連線當基線，中間
   最遠點的 **垂距 ÷ 弦長 > 0.25（約 25%）** 才保留該點，再對左右兩半遞迴。
3. 保留的中段頂點**本身必須是黑點站**才標粉紅（不是站的幾何頂點不標）。
4. 同一黑點出現在多條共線邊 → 取較大的轉折比例（被任一邊保留即為粉紅）。

常數：`PINK_SINUOSITY = 1.25`、`PINK_DP_TOL = 0.25`（`src/stores/skeleton.js`）。
- **灰＝過長黑點連續段的分隔**（規則）：任兩相鄰「邊界點」之間，黑點最多 **4** 個。
  邊界＝邊的兩端真實節點 ＋ 內部粉紅（灰也成為之後的邊界）。
  一連續段有 **N** 個黑點時，改灰的個數 **G = ⌊N / 5⌋**（4+1=5；N=5→1、N=10→2…）；
  灰點在段內盡量均分（偏中間）。**優先序：已是粉紅的不改灰**。

## 骨架畫面上的點與邊

| 顏色 | 角色 |
|--|--|
| 紅 | 分歧／轉乘（degree≥3，或兩側 route 不同的 degree-2） |
| 藍 | 真端點（degree≤1） |
| 黃 | 幾何交叉新點 |
| 紫 | 頭尾共點／環線切斷點 |
| 黑 | 直通中段站 |
| 粉紅 | 曲折邊的代表性轉折黑點 |
| 灰 | 過長黑點段中間的分隔點 |
| 紅底邊 | 共線合併 |
| 藍邊 | 頭尾共點分歧 |
| 綠邊 | 環線 |

## 使用流程（UI）

1. 先在 Metro Maps 匯入城市（或匯入 GeoJSON）。
2. 在 **Map Adjust** 的 D3.js 視圖選定來源（或直接匯入檔案）。
3. 按 **「骨架化」**（再按一次還原）；可與「依建議旋轉」並用。

## 實作契約（改功能時遵守）

- `buildConnectSkeleton(geojson)` 為**純函式**：不改輸入；回傳
  `{ stationClass: Map<id, 'red'|'blue'|'black'|'purple'|'pink'|'gray'>,
     edges: [{ path:[id…], cls:'coline'|'loop'|'parallel'|'plain', color }],
     yellow: [[lng,lat]…] }`。座標一律取原座標（**不移動、不拉直**）。
- degree／route 集合一律由 `routes[].stations` 建圖得出（不讀 `station_role`）。
- v1 範圍：①③④⑤⑥（節點/邊分類、紫點、粉紅、灰）。②**黃色幾何交叉**列為 v2（metro
  交叉多為轉乘站，黃點罕見）——加時同步更新本文件。

## 修改此轉換時

節點/邊分類、切點、粉紅/灰規則變動，**同步更新本 SKILL.md 與 `src/stores/skeleton.js`**，
使「骨架化的依據」與實作一致。
